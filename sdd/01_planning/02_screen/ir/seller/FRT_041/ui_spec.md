- primary route: `/settings/settlement/edit`
- access: `role:SELLER`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `42`

## Screen Requirements

### 1. 정산 계좌 정보 입력/첨부 영역
  - 본 영역은 정산 계좌 정보의 정합성 확보(입금 실패 방지)
  - 와 소유자 확인(명의 불일치·도용 방지)를 위한 필수 입
  - 력/첨부 블록으로 구성됨.
  - 저장 시점에 아래 입력값/파일이 정산 계좌 정보로 신규
  - 등록 또는 기존 정보로 대체 반영되는 구조이며, 이후 정
  - 산 단계에서는 해당 값이 “현재 유효 계좌”로 참조됨.
  - 예금주명(통장 사본과 일치)
  - 정산 입금 대상 계좌의 예금주 한글/영문명 입력값으
  - 로 사용됨.
  - 입력 가이드는 “통장 사본 기준 표기명과 동일”을 전
  - 제로 하며, 계좌 검증/정산 처리에서 명의 불일치 판
  - 단 기준으로 사용됨.
  - 공란 상태에서는 저장 불가
  - 예금주명이 통장 사본의 표기명과 불일치할 경우, 저
  - 장은 가능하더라도 정산 단계에서 확인 보류/재요청
  - 사유로 전환될 수 있음
  - 은행
  - 정산 계좌의 금융기관 식별값으로 사용됨.
  - 드롭다운 선택 구조이며, 미선택 상태에서는 “은행 선
  - 택” placeholder 유지됨.
  - 선택 완료 시 계좌번호 해석·입금 경로의 기준 값으로
  - 사용되며, 저장 시 정산 계좌 정보에 함께 반영됨.
  - 은행 미선택 상태에서는 저장 불가
  - 계좌번호
  - 정산 입금 계좌의 번호 입력값으로 사용됨.
  - “- 제외하고 입력” 가이드를 기준으로 숫자만 입력되
  - 는 값으로 취급됨.
  - 공란 상태에서는 저장 불가
  - 입력값은 저장 시 표준 형식(하이픈 제거된 형태)으로
  - 저장되는 기준값으로 취급되며, 이후 정산 처리 시 실
  - 입금 계좌 식별자로 참조됨.
  - 통장 사본(파일 업로드 전 UI)
  - 통장 사본 첨부를 위한 업로드 전 상태 UI가 노출됨.
  - 업로드 전에는 드롭존/업로드 버튼 중심의 안내 UI가
  - 유지되며, 파일 미첨부 상태에서는 증빙 미완료 상태
  - 로 판단됨.
  - 허용 파일 형식 및 용량 제한(PNG/JPG/PDF, 5MB
  - 이하)이 안내 문구로 고지되며, 조건 불충족 파일 선
  - 택 시 업로드 불가 처리 전제로 취급됨.
  - 통장 사본은 예금주명·계좌번호의 증빙 근거로 사용
  - 되며, 저장 시점에 함께 제출되지 않으면 저장 불가
  - 신분증 사본(파일 업로드 후 UI)
  - 업로드 완료 상태에서는 업로드된 파일이 “현재 제출
  - 된 신분증”으로 취급되며, 저장 시 해당 파일 참조값
  - 이 정산 계좌 정보에 연결됨.
  - 삭제(×) 동작이 가능한 UI가 포함되어, 삭제 선택 시
  - 신분증 첨부 상태가 미첨부로 전환되고 저장 시 재첨
  - 부가 요구되는 상태로 판단됨.
  - 신분증 사본은 계좌 소유자 확인의 보조 증빙으로 사
  - 용되며, 미첨부 또는 삭제 상태에서는 저장 불가

### 2. 저장
  - ① 영역의 입력값/첨부 상태를 기준으로 정산 계좌 정보
  - 등록/수정 확정 트리거로 동작함.
  - 저장 성공 시, 이후 거래의 정산 단계에서 본 계좌가 입금
  - 대상 계좌로 적용 가능 상태로 전환됨.
