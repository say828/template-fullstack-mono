# FRT_030 UI Spec

- screen code: `FRT_030`
- screen name: `인도정산진행`
- primary route: `/seller/vehicles/:vehicleId/delivery-settlement-progress`
- access: `role:SELLER`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `31`

## Screen Requirements

### 1. 거래 요약 영역
  - 현재 거래 대상 차량의 정보와 최종 합의된 거래 금액이
  - 고정 데이터로 노출됨.
  - 차량명, 연식, 주행거리, 연료 정보, 최종 거래 금액이 변
  - 경 불가 상태로 표시됨..

### 2. 차량 상세 보기 버튼
  - 클릭 시 해당 차량의 최신 상태 기준 차량 상세 화면
  - (FRT_029) 으로 이동됨.
  - 본 화면에서 입력·확정된 진행 상태는 유지되며, 거래 단
  - 계에는 영향 없음.

### 3. 차량 및 서류 인도 영역
  - 오프라인 인도 완료 여부를 판매자가 확인·확정하는 단
  - 계.
  - 판매자는 실제 오프라인 인도가 완료된 이후에만 체크 가
  - 능
  - 차량과 서류를 인도 완료했습니다 체크 시, 인도 완료 상
  - 태가 서버에 확정 저장됨
  - 인도 완료 확정 전에는 이후 단계(5, 6, 7번) 진행 불가
  - 인도 완료 버튼은 인도 체크 완료 상태에서만 활성화되
  - 며, 클릭 시 인도 완료 단계가 거래 흐름상 확정되고 다음
  - 단계로 진행 가능 상태가 됨.

### 4. 인도 완료 버튼
  - 인도 체크가 완료된 경우에만 활성화됨.
  - 클릭 시 인도 완료 상태가 확정되며, 거래 단계 내 “인도
  - 처리 완료” 상태로 기록됨.
  - 이후 단계에서 인도 상태를 되돌릴 수 없음.

### 5. 명의 이전 / 말소 서류 확인 영역
  - 딜러가 인도 완료 이후 업로드한 명의 이전 및 말소 관련
  - 서류가 표시됨.
  - 판매자는 업로드된 파일을 다운로드하여 실물 서류와 대
  - 조 검토
  - 명의 이전 / 말소 서류를 확인했습니다 체크 시 서류 검증
  - 완료 상태 확정
  - 서류 확인 완료 전에는 정산 확인 단계(6번) 진행 불가

### 6. 대금 입금 / 정산 확인 영역
  - 딜러가 PALKA 계좌로 거래 대금을 송금하고, 어드민이
  - 입금 및 수수료 정산을 완료한 이후에만 활성화되는 단
  - 계.
  - 판매자는 실제 입금 완료 사실을 기준으로 계좌 입금(정
  - 산금) 수령을 확인했습니다 체크
  - 체크 시 정산 확인 완료 상태 확정
  - 정산 확인 전에는 거래 완료 처리(7번) 불가

### 7. 거래 완료 처리 버튼
  - 3·5·6번 모든 단계가 확정된 경우에만 활성화됨.
  - 클릭 시 거래 상태가 ‘거래 완료’ 로 최종 확정 되며, 해당
  - 거래는 더 이상 수정·되돌리기 불가
  - 이후 모든 거래 화면은 조회 전용 상태로 전환됨
