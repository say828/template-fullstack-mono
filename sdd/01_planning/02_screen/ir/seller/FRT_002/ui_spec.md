# FRT_002 UI Spec

- screen code: `FRT_002`
- screen name: `로그인(판매자)`
- primary route: `/login?role=seller`
- access: `public`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `3`

## Screen Requirements

### 1. 역할 선택 탭
  - 판매자 로그인 / 딜러 로그인 탭으로 구성됨
  - 기본 선택 상태: 판매자 로그인
  - 사용자가 탭을 전환하면, 해당 선택값이 현재 로그인 시
  - 도 계정의 역할 기준값으로 설정됨
  - 이후 로그인 처리 시 해당 역할 기준으로 계정 검증 및 접
  - 근 권한 판단이 수행됨
  - 계정에 저장된 실제 역할과 선택된 역할이 불일치할 경우
  - 로그인 불가 처리됨
  - 딜러 로그인 탭 클릭 시 (FRT_003) 화면으로 탭 전환

### 2. 이메일 입력 필드
  - 계정 식별용 필수 입력 항목
  - 입력 형식은 이메일 형식만 허용됨
  - 미입력 / 형식 오류 상태에서는 로그인 버튼 비활성화 유
  - 지됨
  - 로그인 실패 시에도 기존 입력값 유지됨

### 3. 비밀번호 입력 필드
  - 계정 인증용 필수 입력 항목
  - 입력값 마스킹 처리됨
  - 미입력 상태에서는 로그인 버튼 비활성화 유지됨

### 4. 자동 로그인
  - 체크 시 로그인 성공 후 다음 접속부터 로그인 절차 생략
  - 가능 상태로 저장됨
  - 체크 해제 상태에서는 매 접속 시 로그인 화면 노출됨

### 5. 로그인 버튼
  - 필수 입력값 충족 시에만 활성화됨
  - 클릭 시 입력된 정보 기준으로 인증 시도
  - 인증 성공 시:
  - 역할 타입 확정
  - 해당 역할 기준 메인 홈(FRT_013) 화면으로 즉시 전
  - 환됨
  - 인증 실패 시:
  - 오류 메시지 노출
  - 현재 화면 유지
  - 입력된 이메일 유지

### 6. 비밀번호 찾기
  - 클릭 시 비밀번호 찾기 화면(FRT_004)으로 이동
  - 비로그인 상태에서도 접근 가능

### 7. 회원가입
  - 클릭 시 회원가입 유형선택(FRT_005) 화면으로 이동

### 8. 소셜 로그인 영역
  - Google / Naver 로그인 제공
  - 소셜 인증 성공 후 계정이 존재하지 않을 경우 회원가입
  - 플로우로 전환됨(FRT_006)
  - 기존 계정 존재 시 해당 서비스 메인(FRT_013)화면으로
  - 전환됨
