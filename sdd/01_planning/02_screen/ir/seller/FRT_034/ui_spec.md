# FRT_034 UI Spec

- screen code: `FRT_034`
- screen name: `등록내용확인`
- primary route: `/seller/vehicles/register/confirm`
- access: `role:SELLER`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `35`

## Screen Requirements

### 1. 등록 내용 요약 영역
  - FRT_033에서 입력된 데이터를 서버 저장 값 기준으로
  - 재구성하여 읽기 전용으로 노출하는 요약 영역이며, 사용
  - 자가 “등록 실행 전 최종 판단”을 할 수 있도록 입력 단위
  - 그대로 묶음 구조로 고정 노출됨.
  - 차량 기본 정보: 대표 이미지 1장 + 제조사/모델/연식/주
  - 행거리/연료/변속기/색상/배기량/차량번호/1인 신조/
  - 사고·수리 이력 값이 편집 불가 텍스트로만 노출됨.
  - → 본 섹션은 “차량 식별/거래 기준 데이터”로 간주되며,
  - 실제 정보 불일치 시 경매 진행 제한 또는 거래 취소 가능
  - 경고 문구가 함께 고정 노출됨.
  - 옵션 및 추가 정보: 선택된 옵션은 선택 칩(태그) 리스트
  - 로 요약 노출되고, 특이사항/셀링포인트는 입력된 본문
  - 이 그대로 읽기 전용 노출됨.
  - → 미입력(선택 없음/내용 없음) 상태는 빈 값으로 저장
  - 된 것이므로, 화면에서는 “없음” 표시됨
  - 차량 사진(대표 이미지 포함): 업로드된 이미지가 썸네일
  - 리스트로 노출되며, 대표 이미지가 목록 선두에 고정 노
  - 출됨.
  - → “+5장 더보기”는 추가 이미지가 존재할 때만 노출되
  - 며, 클릭 시 이미지 뷰어(FRT_015)를 호출함.
  - 입찰 조건: 거래 유형, 입찰 기간, 입찰 시작가, 희망 낙찰
  - 가(참고용), 최소 호가 단위, 딜러 노출 국가, 수출 가능 여
  - 부가 입찰 운영 기준 값으로 요약 노출됨.
  - → 입찰 시작 이후 입찰 시작가/입찰 기간 변경 불가 안
  - 내 문구가 함께 고정 노출되어, “등록 실행 시점이 사실상
  - 확정 시점”임을 명확히 인지시킴

### 2. 이전 버튼
  - 등록 실행 전 단계로 되돌리기 위한 제어 버튼이며, 클릭
  - 시 FRT_033으로 전환됨.
  - 전환 시점에 본 화면은 입력 불가 화면이므로 별도 저장
  - 은 발생하지 않으며, 수정은 FRT_033에서만 가능함.

### 3. 경매 등록하기 버튼
  - 판매자가 본 화면의 내용을 “확정 동의”한 것으로 간주되
  - 는 최종 실행 버튼이며, 클릭 시 경매 등록 요청이 서버에
  - 제출됨.
  - 제출 성공 시, 차량은 “입찰대기” 상태로 전환되며 내차량
  - 목록 최상단에 반영됨
  - 이후 단계에서 입찰 시작이 도래하면 변경 불가 조건이
  - 적용됨.
- 차량 등록이 완료되었습니다.
  - 차량이 성공적으로 등록되었습니다.
- 지금부터 딜러들의 입찰이 시작됩니다.
  - 확인
