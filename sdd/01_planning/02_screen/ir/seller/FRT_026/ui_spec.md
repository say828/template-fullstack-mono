- screen code: `FRT_026`
- screen name: `감가재협의요청`
- primary route: `/seller/vehicles/:vehicleId/depreciation/renegotiation`
- access: `role:SELLER`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `27`

## Screen Requirements

### 1. 감가 기준 정보 영역
  - 기존 낙찰가, 딜러 최종 제안 금액이 서버 기준 감가 판단
  - 기준값으로 고정 로드됨.
  - 해당 값은 본 재협의 요청의 비교 기준 데이터로 사용되
  - 며, 재협의 요청 완료 전까지 수정 불가 상태 유지됨.
  - 본 값은 이후 감가 협의 확정 시 거래 금액 및 정산 구조
  - 계산의 기준 원본 데이터로 사용됨.

### 2. 딜러 감가 제안 내역 영역
  - 딜러가 등록한 감가 항목·사유·금액 데이터가 서버 기준
  - 그대로 조회 노출됨.
  - 해당 데이터는 감가 협의 판단 근거 원본으로 저장되며,
  - 본 화면에서는 수정 불가 상태 유지됨.
  - 재협의 요청 완료 시, 해당 감가 내역은 “이전 제안” 이력
  - 으로 보존됨.

### 3. 재협의 요청 입력 영역
  - 판매자가 감가 제안에 대한 이의 사유(텍스트), 희망 조정
  - 금액(숫자), 증빙 파일(첨부)을 입력함.
  - 모든 입력값은 재협의 요청 데이터로 서버에 신규 생성
  - 저장됨.
  - 재협의 요청 데이터는 딜러 재검토 판단 기준값으로 전달
  - 되며, 딜러 화면에 재협의 요청 알림이 즉시 생성됨.
  - 재협의 요청 접수 이후, 기존 감가 제안은 확정 보류 상태
  - 로 전환됨.

### 4. 취소 버튼
  - 클릭 시 본 화면에서 입력된 재협의 요청 내용을 저장하
  - 지 않고 이전 감가 협의 화면으로 복귀함
  - 거래 상태 및 감가 협의 상태는 변경되지 않음.

### 5. 재협의 요청 버튼
  - 클릭 시 입력된 모든 재협의 데이터가 서버에 저장되고,
  - 해당 거래의 감가 협의 상태는 ‘재조정 대기’ 로 변경됨
  - 동시에 딜러에게 재협의 요청 알림이 발송되며, 딜러 감
  - 가 재검토 단계로 거래가 전환됨..
