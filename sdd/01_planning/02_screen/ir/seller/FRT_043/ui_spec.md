## Screen Requirements

### 1. 비밀번호 변경 영역
  - 계정 인증 수단인 로그인 비밀번호를 갱신하는 핵심 보안
  - 제어 영역.
  - 입력 필드 간 검증 결과에 따라 변경 가능 여부가 판단된
  - 다.
  - 기존 비밀번호
  - 현재 계정에 저장된 비밀번호와의 일치 여부 검증 기
  - 준 입력값.
  - 불일치 시 변경 절차 진행 차단.
  - 새 비밀번호
  - 신규 로그인 비밀번호 후보값 입력 필드.
  - 영문 대소문자, 숫자, 특수문자 중 2가지 이상 조합 +
  - 10~16자 조건 충족 필수.
  - 조건 불충족 시 변경 요청 불가.
  - 새 비밀번호
  - 신규 로그인 비밀번호 후보값 입력 필드.
  - 영문 대소문자, 숫자, 특수문자 중 2가지 이상 조합 +
  - 10~16자 조건 충족 필수.
  - 조건 불충족 시 변경 요청 불가.

### 2. 비밀번호 변경 실행
  - ① 영역의 모든 검증 조건 충족 시 활성화.
  - 클릭 시 현재 세션 계정의 인증 비밀번호 갱신 요청 확정
  - 트리거로 동작.
  - 변경 완료 즉시 이후 로그인 인증 기준값이 신규 비밀번
  - 호로 전환됨

### 3. 로그인 보안 설정 영역
  - 로그인 환경 보안을 보조적으로 강화하는 정책 제어 영
  - 역.
  - 새 기기 로그인 알림
  - 새로운 기기에서 로그인 시도 발생 여부 감지 후 이메
  - 일 알림 발송 여부 판단 기준값.
  - ON 상태 시 보안 이상 감지 알림 활성화.
  - 장기 미접속 시 자동 로그아웃
  - 지정 시간 이상 활동 없을 경우 세션 만료 처리 기준
  - 값.
  - 선택 값에 따라 계정 세션 자동 종료 시간 정책 갱신.
  - 본 영역 설정은 계정별 세션 유지 정책 데이터로 저장
  - 되며, 이후 모든 로그인·세션 관리 판단 기준으로 적
  - 용됨
- 새 비밀번호로 다시 로그인해 주세요.
  - 확인
