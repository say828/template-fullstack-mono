# FRT_004 UI Spec

- screen code: `FRT_004`
- screen name: `비밀번호찾기`
- primary route: `/forgot-password?role=seller`
- access: `public`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `5`

## Screen Requirements

### 1. 이메일 주소 입력
  - 비밀번호 재설정 대상 계정을 식별하기 위한 필수 입력
  - 항목으로 구성됨.
  - placeholder 문구는 “가입하신 이메일을 입력하세요”로
  - 고정 노출됨.
  - 입력값은 이메일 형식만 허용되며, 형식 오류 상태에서는
  - 하단 실행 버튼이 비활성화 유지됨.
  - 사용자가 입력을 완료하더라도, 계정 유형이 선택되지 않
  - 은 상태에서는 실행 버튼이 활성화되지 않음.
  - 요청 실패/성공 여부와 무관하게, 사용자가 입력한 이메
  - 일 값은 화면 내 유지됨

### 2. 계정 유형 선택(라디오)
  - 판매자 계정 / 딜러 계정 라디오 버튼으로 구성됨.
  - 기본 선택값은 판매자 계정으로 설정됨
  - 사용자가 계정 유형을 변경하면 해당 선택값이 재설정 요
  - 청의 기준 역할로 적용됨.
  - 가입 이메일이 동일하더라도 계정 유형에 따라 조회 대상
  - 이 달라지므로, 계정 유형 선택은 필수값으로 취급됨.
  - 계정 유형과 이메일 조합이 유효하지 않을 경우(해당 유
  - 형으로 가입 이력 없음) 요청은 실패 처리됨.

### 3. 비밀번호 찾기(실행 버튼)
  - 이메일 형식 유효 + 계정 유형 선택 완료 조건에서만 활
  - 성화됨.
  - 클릭 시 재설정 요청이 접수되며, 요청 성공 시 사용자에
  - 게 “이메일 발송 완료” 상태가 인지되도록 후속 안내가 제
  - 공
  - 요청 성공 이후에는 로그인 화면으로 복귀

### 4. 로그인 화면으로
  - 클릭 시 로그인 화면으로 즉시 이동함.
  - 현재 입력 중이던 정보는 폐기되며, 로그인 화면 초기 상
  - 태로 재진입됨.
  - 로그인 화면으로 복귀 시 판매자/딜러 탭은 이전 사용자
  - 가 마지막으로 선택했던 역할 상태를 유지

### 5. 회원가입 하기
  - 클릭 시 회원가입 플로우(FRT_005)로 이동함.
  - 로그인/재설정 흐름에서 “계정이 존재하지 않음” 상황이
  - 발생했을 때 사용자가 즉시 가입으로 전환할 수 있는 보
  - 조 진입점 역할을 수행함.
