# FRT_036 UI Spec

- screen code: `FRT_036`
- screen name: `정산완료상세`
- primary route: `/seller/settlement/completed/:vehicleId`
- access: `role:SELLER`
- source spec: `sdd/01_planning/02_screen/ir/seller/seller_screen_spec.pdf`
- page number: `37`

## Screen Requirements

### 1. 정산 완료 상태 패널
  - 해당 거래가 정산 완료 상태로 최종 확정되었음을 상단에
  - 서 고정 표시.
  - 정산 금액이 판매자 계좌로 입금 완료 상태임을 함께 명
  - 시적으로 노출.
  - 본 패널 노출 시점 기준, 해당 거래는 완전 종결 상태로 고
  - 정되며 이후 단계 전환·되돌림 불가 상태 유지됨.
  - 본 영역은 정산 상태 = 완료인 거래에서만 노출됨

### 2. 거래 기본 정보 영역
  - 해당 거래의 최종 결과 식별 정보 블록으로 구성됨.
  - 거래번호, 거래 완료일, 차량 대표 이미지, 차량명, 연식,
  - 주행거리, 연료 타입, 거래 유형(국내/수출), 거래 국가 정
  - 보가 거래·정산·서류 절차 종료 시점 기준의 확정 데이터
  - 로 고정 노출됨.
  - 본 영역의 정보는 이미 종료된 거래와 정산 결과를 검증·
  - 증빙하기 위한 기준 데이터로 사용되며, 본 화면 진입 시
  - 점 기준 모든 항목은 수정·변경·삭제 불가 상태로 유지됨.

### 3. 정산 내역 상세 영역
  - 본 거래의 정산 결과 산식 구조 검증 영역으로 구성됨.
  - 최종 거래 금액이 기준값으로 노출되고, 그 하위로 성능
  - 검사비, 탁송비, 중개수수료(VAT 포함) 항목이 각각 분리
  - 노출됨.
  - 각 항목 차감 결과로 도출된 정산 금액(판매자 수령액) 이
  - 최종 확정값으로 표시됨.
  - 본 영역의 모든 금액 데이터는 정산 완료 시점 기준 확정
  - 값으로 고정됨

### 4. 명의 이전 / 말소 서류 영역
  - 거래 완료 이후 등록된 법적 이전·말소 이력 관리 영역으
  - 로 구성됨.
  - 명의 이전·말소 관련 파일 목록이 업로드 완료 상태로 노
  - 출되며, 각 파일 다운로드 가능 상태 유지됨.
  - 모든 필수 서류 등록 완료 시 “모든 서류가 정상 등록되었
  - 습니다.” 상태 메시지 고정 노출됨
  - 본 영역의 이력 데이터는 정산 완료 이후 영구 보존 이력
  - 데이터로 유지되며 수정·삭제 불가 상태임

### 5. 정산 요약 영역
  - 거래 전체 정산 결과를 요약 검증하는 영역으로 구성됨.
  - 최종 거래 금액, 총 공제 금액, 정산 금액(입금 완료) 이 요
  - 약 정보로 노출됨.
  - 입금 완료 일시가 함께 표시되며, 해당 시점을 기준으로
  - 정산 상태는 최종 종료 상태로 확정됨.
  - 본 영역 데이터는 3번 정산 내역과 동일 기준 값으로 고
  - 정 연동됨.

### 6. 입금 / 계좌 정보 영역
  - 정산 금전 흐름의 최종 확정 정보 블록으로 구성됨.
  - 정산 상태 입금 완료 고정 표시, 정산 방식(국내 계좌 이
  - 체 / 해외 송금 방식 등) 거래 유형에 따라 표시됨.
  - 판매자 계좌 정보(은행명, 계좌번호, 예금주)가 확정값으
  - 로 노출됨.

### 7. 정산 명세서 다운로드 영역
  - 본 거래의 정산 결과를 증빙 문서 형태로 보관·출력하기
  - 위한 기능 영역.
  - 정산 명세서는 거래·정산·공제·입금 정보가 모두 반영된
  - 최종 확정 문서이며, PDF 형태로 다운로드 가능
