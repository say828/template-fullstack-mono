## Screen Requirements

### 1. 알림 채널 설정 영역
  - 사용자의 알림 전달 수단 선호도를 설정하는 영역.
  - 각 체크박스는 독립적 다중 선택 구조로 동작하며, 최소 1
  - 개 이상 선택 상태 유지 전제.
  - 이메일로 알림 받기
  - 거래/정산/검사/인도 관련 이벤트 발생 시 이메일 발
  - 송 대상 포함 여부 판단 기준 값.
  - 체크 해제 시 이메일 채널을 통한 모든 알림 전송 제
  - 외 상태로 전환.
  - 문자(SMS)로 알림 받기
  - 거래 주요 이벤트 발생 시 SMS 발송 대상 포함 여부
  - 판단 기준 값.
  - 미체크 시 SMS 채널 알림 전송 전면 차단 상태로 전
  - 환.
  - 웹 알림(브라우저) 받기
  - 서비스 접속 중 브라우저 알림 노출 여부 판단 기준
  - 값.
  - 미체크 시 실시간 웹 알림 미노출 상태로 전환.
  - 본 영역의 체크 상태는 알림 채널 정책 데이터로 저장
  - 되며, 이후 모든 알림 전송 로직에서 전송 경로 필터
  - 기준으로 사용된다

### 2. 알림 유형 설정 영역
  - 거래 단계별 이벤트 발생 시 알림 허용 여부를 세분화하
  - 여 통제하는 영역.
  - 각 토글은 개별 유형 단위 활성/비활성 구조로 동작.
  - 각 유형의 ON/OFF 상태는 이벤트 유형별 알림 허용 정
  - 책 데이터로 저장되며, 알림 발생 시 채널 설정(①)과 함
  - 께 교차 검증 기준으로 사용됨

### 3. 저장
  - ①, ② 영역의 현재 상태를 판매자 알림 정책 데이터로 최
  - 종 확정하는 트리거.
  - 저장 시점에 해당 계정의 알림 설정 값이 전면 갱신됨.
  - 저장 완료 후, 이후 발생하는 모든 이벤트 알림은 본 화면
  - 에서 저장된 설정값 기준으로 전송/차단 판단이 수행됨
