# ADM_011 UI Spec

- screen code: `ADM_011`
- screen name: `거래상세(검차)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `12`

## Screen Requirements

### 1. 검차 일정 영역
  - 일정 상태
  - 값: 확정 / 미등록
  - 검차 일정이 저장 완료되면 확정으로 고정 노출됨
  - 확정 이후 일정 수정 발생 시 최신 일정 갱신 구조 유
  - 지
  - 검차 일시
  - 기준: 검차 일정 등록 시 입력된 검차 예정 시각
  - 이후 변경 불가 기준값
  - 감가 단계 진입 조건 판단 기준 필드
  - 검차 장소
  - 검차 일정 등록 시 입력된 위치 정보
  - 인도 단계 일정 산정 시 참조값으로 사용
  - 검차사 / 연락처
  - 검차 일정 등록 시 선택/입력된 검차사 정보
  - 거래 분쟁 및 이력 검증 시 기준 데이터로 유지
  - 데이터 미등록 상태
  - 검차 일정 미등록 시 모든 필드는 빈값 + 안내 문구 형
  - 태로 노출
  - 다음 단계 진입 차단 조건으로 작동

### 3. 검차 결과 영역
  - 외관 / 엔진·미션 / 사고 이력 / 침수 이력 / 주행 성능
  - 각 항목은 검차 리포트 기준 판정값 노출
  - 값 기준: 정상 / 이상 / 없음 / 확인 불가 등 정책 코드
  - 값
  - 첨부 파일
  - 검차 완료 시 업로드된 검차 리포트 원본 파일
  - 이후 단계에서 분쟁 증빙 자료로 사용되는 기준 문서
  - 검차 결과 미등록 상태
  - 결과 미등록 시 모든 항목은 빈값
  - 감가 단계 진입 차단 조건으로 작동

### 4. 검차 리포트 상세보기 버튼
  - 클릭 시 검차 운영 상세(ADM_024) 화면으로 이동
