# ADM_058 UI Spec

- screen code: `ADM_058`
- screen name: `관리자계정관리`
- primary route: `/admin/accounts`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `59`

## Screen Requirements

### 1. 검색(이름/이메일)
  - 입력값 기준 부분 일치 검색.
  - 검색 대상: 이름, 이메일 2개 필드.
  - 검색 결과 반영 범위: 4번 목록 전체.
  - 공백/미입력 상태: 필터(2번)만 적용된 기본 목록 노출.

### 2. 필터 영역(권한 그룹 / 계정 상태)
  - 권한 그룹 필터
  - 선택값 기준 그룹 소속 계정만 노출.
  - 전체 선택 시 권한 그룹 조건 미적용.
  - 계정 상태 필터
  - 전체 / 사용 / 사용중지 3상태로 목록 노출 범위 제어.
  - 사용: 로그인 및 운영 기능 접근 허용 상태의 계정만
  - 노출.
  - 사용중지: 로그인 및 운영 기능 접근 차단 상태의 계
  - 정만 노출.
  - 필터 조합 적용
  - 권한 그룹 × 계정 상태 × 검색어(1번) 동시 적용.
  - 결과 건수(총 n건) 표기는 현재 조건의 결과 기준으로
  - 동기화

### 3. 계정 등록
  - 신규 관리자 계정 생성 버튼
  - 클릭 시 관리자 계정 등록 화면(ADM_059)을 호출함
  - 마스터 권한 관리자만 노출되는 버튼으로 제한 가능

### 4. 관리자 계정 목록
  - 리스트 단위: 운영자 계정 1개 = 1행.
  - No: 목록 내 정렬 기준에 따른 노출 순번.
  - 관리자ID: 운영 계정 고유 식별자(중복 불가 전제)
  - 이름: 계정 소유자 표시값.
  - 이메일: 로그인/회신/초기화 발송의 기준 주소.
  - 권한 그룹: 운영 권한 묶음의 소속 값(권한 범위 판단
  - 기준).
  - 계정 상태: 사용/사용중지 태그로 즉시 식별, 접근 통
  - 제 여부의 결과값.
  - 마지막 접속 일시: 최근 성공 접속 시각
  - 생성일: 계정 생성 시각
  - 비밀번호 초기화: 5번 액션 트리거 영역.
  - 리스트행 클릭 시 해당한은 관리자의 상세화면
  - (ADM_059)을 호출함

### 5. 비밀번호 초기화 버튼
  - 비밀번호를 초기화하여 새로운 임시 비밀번호를 생성
  - 클릭 시 서버로 비밀번호 초기화 요청 전송
  - 초기화 후 사용자는 임시 비밀번호로 로그인한 뒤 새로운
  - 비밀번호를 설정해야 함
  - 비밀번호 초기화 버튼은 상세 화면으로 진입 시 표시되며
  - 등록화면으로 진입 시 히든 처리
  - 비밀번호초기화 버튼 클릭 시 해당 유저의 이메일로 임시
  - 비밀번호 메일 발송 처리
