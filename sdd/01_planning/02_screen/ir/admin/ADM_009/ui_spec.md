# ADM_009 UI Spec

- screen code: `ADM_009`
- screen name: `거래상세(입찰대기)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `10`

## Screen Requirements

### 1. 상단 거래 식별 영역
  - 거래 고유 ID(TR-20250220) 표시.
  - 해당 거래 관리 상위 경로 유지.
  - 본 화면은 항상 단일 거래 단위 기준으로 동작
  - ← 버튼 클릭 시 현재 거래의 요약 패널 상태로 복귀.

### 2. 거래 화면보기 버튼
  - 현재 선택된 거래 건을 판매자/딜러 등 실제 사용자에게
  - 노출되는 프론트 거래 상세 화면 기준으로 확인하기 위한
  - 이동버튼
  - 클릭 시 어드민 화면과는 별도의 새창으로 해당 거래의
  - 프론트 거래 상세 페이지가 호출되며, 관리자는 실제 사
  - 용자 관점에서 노출상태, 단계 흐름 등을 확인할 수 있음

### 3. 거래자 정보 종료 버튼
  - 클릭 시 해당 거래의 현재 단계 무관 강제 종료 처리하기
  - 위한 강제종료(ADM_017) 화면을 호출함
  - 종료 시 거래 상태 취소로 확정 전환.(ADM_018 참고)
  - 종료 즉시 이후 단계 전환 불가.

### 4. 거래 상세 정보 영역
  - 차량 정보: 거래 대상 차량의 고유 식별 정보로, 해당 거래
  - 가 어떤 차량을 기준으로 생성되었는지를 판단하는 기준
  - 데이터
  - 거래 관계자: 판매자, 담당 딜러를 의미하며 거래 진행의
  - 주체 식별 기준
  - 현재 상태: 거래 메인 상태값이며 단계 탭 활성 여부, 버튼
  - 노출 조건, 알림 트리거 기준으로 사용됨
  - SLA: 현재 단계 기준 잔여 처리 허용 기한으로, 경과 시
  - 관리자 알림 및 리스크 플래그 생성 기준
  - 담당 관리자: 해당 거래의 운영 책임자 계정

### 5. 단계 탭 영역
  - 입찰, 검차, 감가, 인도, 송금·정산, 상태 이력 탭은 모두 클
  - 릭 가능
  - 아직 해당 단계 데이터가 생성되지 않은 경우 빈 화면으
  - 로 표시됨
  - 단계별 데이터는 거래 상태값 전환 시점에 생성됨
  - 현재 거래는 입찰 단계로 확정 표시됨

### 6. 입찰 진행 현황 영역
  - 참여 딜러 수: 유효 입찰 레코드 기준 distinct 딜러 수
  - 현재 최고 입찰가: 유효 입찰 중 최대 가격
  - 평균 입찰가: 유효 입찰가의 산술 평균
  - 입찰 시작: 최초 입찰 레코드 생성 시각
  - 입찰 마감 예정: 거래 마스터에 저장된 종료 시각
  - 잔여 시간: 마감 예정 시각과 현재 시각의 차이
  - 마감 시각 도래 시 낙찰자 확정 프로세스 자동 트리거 발
  - 생

### 7. 입찰 참여 내역 영역
  - 딜러명: 입찰 참여 주체 식별 기준
  - 입찰가: 해당 딜러가 제시한 가격
  - 입찰 시각: 가격 우선순위 판단 보조 기준
  - 국가: 해외/국내 거래 분기 및 수출 로직 판단 기준
  - 상태: 최고가 여부 및 유효 여부 판단 결과
  - 최고 입찰 단일 행만 Highest 상태 유지
  - 입찰 마감 시 판매자가 선택한 딜러가 낙찰 대상자로 확
  - 정됨
