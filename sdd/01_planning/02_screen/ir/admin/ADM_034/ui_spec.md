# ADM_034 UI Spec

- screen code: `ADM_034`
- screen name: `송금관리상세`
- primary route: `/admin/remittances/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `35`

## Screen Requirements

### 1. 헤더(거래 식별/복귀)
  - 거래 단위 컨텍스트를 고정하고 목록 복귀 동선을 제공하
  - 는 영역.
  - 화면 타이틀은 “딜러 송금 관리”로 고정되며, 우측에
  - #TR-YYYYMMDDNNN 형태의 거래 식별자가 함께 노출
  - 됨.
  - 좌측 뒤로가기 클릭 시 딜러 송금 관리 목록으로 복귀되
  - 며, 이전 목록의 필터/페이지 상태 복원

### 2. 거래 화면 보기(프론트 상세 새창)
  - 관리자가 동일 거래를 사용자 관점(프론트)으로 교차 검
  - 증하기 위한 외부 진입 액션.
  - 클릭 시 해당 거래의 프론트 거래 상세 페이지가 새 창으
  - 로 호출됨.

### 3. 거래 정보(송금 확인 컨텍스트 고정)
  - 송금 검증의 기준이 되는 거래/주체/상태/지표를 요약하
  - 는 고정 영역.
  - 거래ID: 내부 거래 식별자. 딜러 송금 내역/확인 이력의
  - 조인 키로 사용됨.
  - 차량 정보: 거래 기준 차량 요약 스냅샷(모델 등). 송금 대
  - 상 거래의 오인 방지 목적.
  - 판매자/딜러: 거래 당사자 표시값. “누가 송금해야 하는
  - 지” 판단의 기준 주체로 사용됨.
  - 송금 금액: 딜러가 송금해야 하는 확정 금액(₩). 이후 섹
  - 션(송금 내역)의 “등록 금액”과 대조 검증 기준으로 사용
  - 됨.
  - 송금 일자: 거래 기준 송금일(운영 기준일). 딜러 제출값
  - 과 서버 기준값이 상이할 수 있어 “거래 정보의 기준 일
  - 자”로 고정됨(정책 통일 필요).
  - 송금 상태: 확인 대기 / 송금 완료 태그로 노출되며, 하단
  - 확정 버튼(6) 노출 조건의 기준값으로 사용됨.
  - SLA: 경과 지표(D+0 등). 기준 시점은 “송금 확인 요청
  - 발생 시각”

### 4. 송금 내역(딜러 제출 정보/증빙 조회)
  - 딜러가 “송금 확인 요청”을 수행하며 입력·첨부한 값을 원
  - 본 그대로 조회하는 영역. 본 영역 데이터는 관리자가 수
  - 정하지 않으며, 확정 판단의 근거로만 사용됨.
  - 송금 일자: 딜러가 제출한 송금일. 저장은 YYYY-MM-DD
  - 단일 값 기준이며, 거래 정보(3)의 기준 일자와 불일치 시
  - 운영자가 이상 징후로 판단할 수 있도록 원본값 유지됨.
  - 송금 금액: 딜러가 제출한 송금 금액(₩). 거래 정보(3)의
  - 확정 송금 금액과 대조되는 검증 포인트로 사용됨.
  - 송금 방법: 제출된 송금 수단(예: 계좌이체). 허용 값은 정
  - 책 코드셋으로 관리되며, 화면은 라벨로 조회됨.
  - 송금 증빙 파일: 딜러가 첨부한 증빙 파일 메타(존재 여
  - 부/파일명 등)와 “송금 증빙 보기” 액션으로 구성됨.
  - “송금 증빙 보기” 클릭 시 파일 다운로드로 연결되며,
  - 파일 접근 권한은 어드민 권한 기반으로 제한됨.

### 5. 송금 확인(관리자 확정 결과)
  - 관리자가 (6)에서 확정 처리했을 때만 채워지는 “확정 결
  - 과” 영역. 확인 대기 상태에서는 비어 있는 값이 기본으로
  - 유지됨.
  - 송금 확인일: 관리자가 확정 처리한 일시. 저장은 서버 타
  - 임스탬프(예: YYYY-MM-DD HH:MM:SS) 기준이며, 화
  - 면은 정책 포맷으로 노출됨.
  - 송금 확인 상태: 확인 대기 / 송금 완료로 노출되며, 거래
  - 정보(3)의 송금 상태와 1:1로 일치해야 함
  - 송금 확인자: 확정 처리한 관리자 계정 표시명. 저장은 관
  - 리자 식별자 + 표시명 스냅샷
  - 송금 완료 상태에서는 본 영역이 확정 정보로 고정 노출
  - 되며, 이후 변경 불가 상태로 유지됨.

### 6. 송금 확인(확정 처리 버튼)
  - 관리자가 “입금 확인”을 최종 확정하는 실행 버튼.
  - 노출 조건은 송금 상태 = 확인 대기인 경우에만 노출됨.
  - 송금 상태 = 송금 완료인 경우 미노출되어 재확정 경로
  - 차단됨.
  - 클릭 시 확정 확인 팝업이 선행되며, 확인 승인 시 아래 처
  - 리가 원자적으로 수행됨.
  - 딜러 송금 상태가 송금 완료로 전환됨.
  - 송금 확인일/확인자 값이 생성되어 (5)에 확정 데이
  - 터로 반영됨.
  - 후속 단계 집계(예: 판매자 정산 처리 대기)에 포함되
  - 도록 거래 파이프라인 상태가 갱신됨(정산 모듈의 선
  - 행 조건으로 사용).
  - 확정 이후 본 화면 재조회 시 (3)(5) 상태가 송금 완료로
  - 일치하고, (6) 버튼은 미노출 상태로 전환됨.
