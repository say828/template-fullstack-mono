# ADM_037 UI Spec

- screen code: `ADM_037`
- screen name: `판매자정산관리상세(완료)`
- primary route: `/admin/settlements/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `38`

## Screen Requirements

### 1. 정산 완료 안내 영역
  - 본 거래가 정산완료 상태로 확정되었음이 상단에 고정 노
  - 출됨.
  - 해당 상태는 실제 정산이 완료되어 판매자에게 지급까지
  - 완료된 거래만 표시 가능하며, 정산이 완료되지 않은 거
  - 래에서는 본 화면 구성 자체가 노출되지 않음.
  - 정산 완료 문구는 단순 상태 표시가 아니라, 해당 거래가
  - 정산 프로세스에서 완전히 종료되었음을 의미하는 최종
  - 확정 표시.

### 2. 거래·정산 기준 정보 영역
  - 정산이 완료된 시점의 모든 거래 및 금액 정보가 스냅샷
  - 기준으로 고정 노출됨. 영역의 정보는 이후 판매자 정보,
  - 계좌 정보, 거래 정보가 변경되더라도 변경되지 않는 기
  - 록임.
  - 차량 정보 : 차량 모델 및 식별 정보가 표시되며, 정산 시
  - 점 기준의 차량 정보가 그대로 유지됨.
  - 판매자 / 딜러 정보 : 정산 당시 거래 당사자인 판매자와
  - 딜러 정보가 노출됨. 이후 사용자 정보가 변경되더라도
  - 본 화면의 정보는 수정되지 않음.
  - 판매자 계좌 정보 : 정산 당시 실제 지급이 이루어진 계좌
  - 정보가 표시됨. 지급 이후 계좌 변경이 발생하더라도 본
  - 화면에는 영향을 주지 않음.
  - 입금 정보 : 딜러 송금이 확인된 시각이 표시되며, 해당 시
  - 각을 기준으로 정산 처리가 이루어졌음을 확인할 수 있
  - 음.
  - 정산 일시 : 정산이 최종 완료된 시각이 별도로 표시됨. 이
  - 값은 본 거래가 정산 단계에서 종료된 시점을 의미.
  - 정산 금액 상세 : 송금 확인 금액, 플랫폼 수수료, 최종 정
  - 산 금액이 모두 노출됨. 수수료는 정책 기준에 따라 계산
  - 된 확정값이며, 최종 정산 금액은 판매자에게 실제 지급
  - 된 금액. 모든 금액은 정산 완료 시점의 확정 기준값으로
  - 고정 유지.

### 3. 확인 버튼 영역
  - 확인 버튼은 본 팝업을 종료하는 기능만 수행.
  - 정산 완료 상태에서는 어떠한 추가 처리 버튼도 존재하지
  - 않으며, 본 거래에 대해 관리자가 수행할 수 있는 후속 동
  - 작은 없음.
