# ADM_030 UI Spec

- screen code: `ADM_030`
- screen name: `인도관리상세(일정대기)`
- primary route: `/admin/deliveries/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `31`

## Screen Requirements

### 1. 거래 식별 영역
  - 선택된 거래의 인도 관리 기준 정보 고정 노출.
  - 거래 ID 기준으로 인도 일정, 인도 확인, 증빙 파일, 승인
  - 이력이 전부 귀속되며, 본 영역의 정보는 본 패널 전 처리
  - 흐름의 기준 키로 사용됨

### 2. 인도 일정 정보 영역
  - 현재 거래에 인도 일정 레코드 미존재 상태
  - “미정” 표시는 인도 일정 테이블에 해당 거래 ID 기준 일
  - 정 데이터가 생성되지 않았음을 의미.
  - 본 상태에서는 인도 프로세스가 다음 단계로 전환 불가.
  - 인도 일정은 딜러가 판매자와 유선 합의 후 수동 등록하
  - 는 구조이며 자동 생성·제안 로직 없음.

### 3. 인도 완료 확인 영역
  - 판매자 확인 / 딜러 확인 모두 비활성 상태 고정.
  - 인도 일정 레코드가 생성되지 않은 상태에서는 인도 완료
  - 판단 자체가 불가한 구조로 처리됨
  - 본 영역은 인도 일정 확정 이전까지 입력·전환·승인 불가
  - 상태 유지.

### 4. 인도 증빙 영역
  - 차량 인도 사진 및 서류 인수 확인서 모두 미등록 상태 고
  - 정.

### 5. 인도 완료 승인 영역
  - 인도 완료 승인 버튼 비활성 상태 고정.
  - 활성 조건은 세 조건 동시 충족 시에만 성립: (인도 일정
  - 레코드 존재 / 딜러 인도 완료 확인 완료 /판매자 인도 완
  - 료 확인 완료)
  - 현재 상태에서는 위 조건 전부 미충족 상태로 승인 불가
