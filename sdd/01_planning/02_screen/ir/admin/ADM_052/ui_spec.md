# ADM_052 UI Spec

- screen code: `ADM_052`
- screen name: `공지사항등록`
- primary route: `/admin/support/notices`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `53`

## Screen Requirements

### 1. 공지 작성 영역(제목/본문/노출 정보)
  - 공지 1건의 확정 데이터를 구성하는 입력 구간.
  - 제목 입력
  - 공지 목록의 ‘제목’ 컬럼 및 공지 상세 상단 제목으로
  - 동일 노출되는 대표 텍스트.
  - 제목 미입력 상태에서는 저장 확정 불가로 취급(필수
  - 값).
  - 본문 입력(서식 편집)
  - 사용자 공지 상세에서 그대로 렌더링되는 확정 콘텐
  - 츠.
  - 줄바꿈/목록/강조/링크/이미지(아이콘 존재) 등 서식
  - 요소는 편집 결과 그대로 유지되는 것을 기본값으로
  - 취급.
  - 노출 채널(드롭다운)
  - 공지를 노출할 사용자 집합을 결정하는 최상위 분기
  - 값.
  - 선택값에 따라 사용자 공지 목록에 포함되는 대상이
  - 결정됨(전체/딜러/판매자).
  - 채널 미선택 상태 저장 불가(필수값)로 취급.
  - 카테고리(드롭다운)
  - 사용자 공지 화면에서의 분류 기준.
  - 채널과 결합되어 사용자 측 필터/탭 구성에 영향
  - 노출 여부(라디오)
  - ‘노출’ 확정 시 즉시 사용자 공지 목록의 노출 대상 포
  - 함.
  - ‘미노출’ 확정 시 사용자 측에서는 접근 불가 상태로
  - 유지되며, 어드민 목록에서는 관리 대상으로만 유지
  - 등록/수정 모드 공통 처리
  - 등록 모드: 모든 입력값 공란 초기화(필수값 미선택
  - 상태).
  - 수정 모드: 기존 공지의 확정값이 그대로 채워진 상태
  - 로 진입, 변경된 값만 저장 시 반영.
  - 수정 모드에서 ‘작성자/수정일’은 목록에서 별도 갱신
  - 됨

### 2. 취소
  - 편집 종료 및 직전 화면(공지 목록 또는 공지 상세) 복귀
  - 트리거.
  - 취소 시 입력 중 변경사항은 확정되지 않은 것으로 취급.

### 3. 저장
  - 공지 등록/수정의 확정 트리거.
  - 저장 확정 시 공지 데이터가 최종 반영되고, 공지 목록/상
  - 세 및 사용자 공지 화면 노출 대상이 갱신됨.
  - 저장 완료 후 공지 상세 화면으로 전환
