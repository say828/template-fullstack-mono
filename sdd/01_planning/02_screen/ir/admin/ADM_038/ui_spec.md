# ADM_038 UI Spec

- screen code: `ADM_038`
- screen name: `판매자관리`
- primary route: `/admin/sellers`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `39`

## Screen Requirements

### 1. 상단 조회 조건(필터/검색) 영역
  - 본 화면의 조회 대상은 “판매자 계정” 단위로 고정됨.
  - 본 영역에서 확정되는 조회 조건은 목록(2번) 데이터/총
  - 건수/페이지 이동 유지 조건으로 동작 범위가 고정됨.
  - 상태 필터(전체/정상/블랙리스트/탈퇴)
  - “판매자 계정의 운영 상태” 값 기준 필터링.
  - 전체: 상태 무관 전체 포함.
  - 정상: 서비스 이용 제한이 없는 계정만 포함.
  - 블랙리스트: 운영에 의해 제한 상태로 분류된 계정만
  - 포함
  - 탈퇴: 탈퇴 완료 처리된 계정만 포함(조회 목적의 이
  - 력만 유지, 운영 액션 대상에서 제외되는 상태 전제).
  - 기본 진입 조건은 “전체”로 고정됨.
  - 기간 필터(오늘/최근7일/최근30일/전체)
  - 기준 일자는 가입일로 고정됨.
  - ‘전체’ 선택 시 기간 조건 미적용.
  - 기간과 상태/검색 조건은 교차 적용되어 최종 조회 결
  - 과가 결정됨.
  - 검색 입력(판매자ID/판매자명/연락처)
  - 검색 키는 3종 중 어느 값이 입력되더라도 동일 입력
  - 창에서 판별되는 구조 전제.
  - 입력값이 숫자/하이픈 포함 형태인 경우 연락처 검색
  - 우선 적용(부분 일치 허용).
  - 입력값이 시스템 식별자 형태(예: 영문+숫자 조합)인
  - 경우 판매자ID 검색 우선 적용.
  - 그 외 문자열은 판매자명 검색으로 처리됨.
  - 검색 조건은 상태/기간 필터와 교차 적용됨.
  - 초기화
  - 상태=전체, 기간=전체, 검색어 공백으로 복원됨.
  - 초기화 직후 목록은 기본 조건(전체/전체/검색없음)
  - 으로 재조회됨.

### 2. 판매자 목록(테이블) 영역
  - 본 화면의 핵심 결과 영역이며, 조회 조건(1번) 확정 결과
  - 를 판매자 단위 목록으로 고정 노출함.
  - 목록 상단의 “총 n건”은 현재 조회 조건이 적용된 결과 건
  - 수로만 계산되어 표시됨(전체 판매자 수와 동일 의미 아
  - 님).
  - 각 행은 “판매자 계정 1건”의 현재 상태를 요약한 스냅샷
  - 으로 취급됨.
  - 각 행 클릭 시 판매자상세(ADM_039) 화면을 호출함
  - 판매자ID
  - 시스템에서 발급된 고유 식별자.
  - 중복 불가, 변경 불가 전제.
  - 판매자명
  - 가입 시점 기반으로 확정된 실명 값.
  - 가입방식
  - 계정 생성 경로 구분 값(이메일/구글/네이버 등).
  - 운영 정책상 “로그인 수단/계정 연결 구조”를 추적하
  - 기 위한 분류 값.
  - 거주국가
  - 판매자 프로필의 국가 값.
  - 국가 값은 거래 가능 권역/정산 가능 통화 정책과 연
  - 동될 수 있으므로 목록에서 기본 노출.
  - 이메일 / 연락처
  - 계정 소유 확인 및 운영 커뮤니케이션 목적의 기본 식
  - 별 정보.
  - “연락처”는 검색 키로도 사용되므로 표기 포맷(하이
  - 픈 포함)은 저장 값과 무관하게 화면 표준 포맷으로
  - 노출되는 전제.
  - 판매 실적
  - 판매자 기준 “완료된 판매 건수” 누적치.
  - 거래 취소/관리자 강제 종료 등 비완료 케이스는 포함
  - 되지 않는 기준 전제
  - 정산계좌(Y/N)
  - 판매자 정산에 필요한 계좌 정보 등록 여부.
  - Y: 정산 처리 가능 상태(계좌 정보가 유효하게 등록되
  - 어 있음).
  - N: 정산 처리 제한 상태(정산 단계 진입 시 보류/확인
  - 필요 사유가 될 수 있음).
  - 본 화면에서는 등록 여부만 판단함
  - 상태(정상/블랙리스트/탈퇴)
  - 판매자의 운영 상태를 단일 값으로 고정 표현.
  - 정상: 거래/정산 등 서비스 핵심 행위 가능 상태.
  - 블랙리스트: 운영 제재 상태로, 거래 생성/진행/정산
  - 등 주요 행위 제한 대상.
  - 탈퇴: 계정 종료 상태로, 신규 행위 불가 및 이력 조회
  - 목적만 남는 상태.
  - 상태 태그 색상은 시각적 구분 목적이며, 필터 조건의
  - 실제 판단 기준은 “상태 값”으로 고정됨.
  - 가입일
  - 계정 생성 확정 일자.
  - 기간 필터의 기준 컬럼과 동일 기준으로 사용됨.
