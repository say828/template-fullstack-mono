# ADM_050 UI Spec

- screen code: `ADM_050`
- screen name: `공지사항관리`
- primary route: `/admin/support/notices`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `51`

## Screen Requirements

### 1. 공지 검색 영역
  - 공지 제목 기준 키워드 검색 영역.
  - 입력 키워드 포함 공지 콘텐츠만 목록 필터링.

### 2. 필터/조회 조건
  - 공지 탐색 조건 제어 영역.
  - 채널: 공지 노출 대상 구분값(전체/판매자/딜러). 선택 값
  - 은 목록(4)의 ‘채널’ 컬럼과 동일 기준으로 적용됨.
  - 카테고리: 공지 성격 분류값(예: 시스템/점검, 이벤트, 업
  - 데이트, 정책/규정, 일반 등). 선택 값은 목록(4)의 ‘카테
  - 고리’ 컬럼 기준으로 필터링됨.
  - 노출여부: 사용자 화면 노출 상태 기준(전체/노출/비노
  - 출). ‘비노출’ 공지는 사용자 화면에 반영 대상에서 제외되
  - 는 상태로 유지되며, 본 목록에서는 관리 대상으로 조회
  - 가능.
  - 초기화: 채널/카테고리/노출여부/검색어 조건을 기본값
  - (전체/전체/전체/빈값)으로 복구.
  - 조건 변경 시 목록(4) 즉시 재조회된 결과로 갱신됨.

### 3. 공지 등록
  - 신규 공지 생성 진입 트리거
  - 클릭 시 ‘공지 등록’ 화면(ADM_052)으로 이동
  - 등록 완료 시 목록(4)에 신규 행 추가됨

### 4. 공지 목록
  - 공지 단위 관리 테이블. 각 행은 1개의 공지 콘텐츠이며,
  - 행 클릭 시 해당 공지의 상세 화면(ADM_051)으로 진입
  - 됨.
  - No: 목록 내 식별용 순번. 정렬 기준 변경 시 재계산되는
  - 표시값으로 취급.
  - 제목: 사용자 화면에 노출되는 공지 제목.
  - 채널: 공지 노출 대상(전체/판매자/딜러).
  - 카테고리: 공지 분류값.
  - 노출여부: 노출/비노출 상태 태그. 태그는 사용자 화면 반
  - 영 여부의 확정 상태를 의미.
  - 등록자: 공지 등록·최종 수정 주체 식별값(관리자 계정).
  - 수정일: 공지 내용 또는 노출 상태가 최종 확정된 시점.
