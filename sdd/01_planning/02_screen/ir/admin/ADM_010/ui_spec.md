# ADM_010 UI Spec

- screen code: `ADM_010`
- screen name: `거래상세(입찰완료)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `11`

## Screen Requirements

### 1. 입찰 요약
  - 입찰 상태는 마감 완료로 확정 노출됨. 해당 값은 거래의
  - 입찰 종료 확정 처리 완료 여부에 의해 결정되며, 확정 이
  - 후 상태값은 되돌림 불가 상태로 유지됨.
  - 마감 완료 시각은 ‘입찰 종료 확정’이 발생한 시각이 아니
  - 라, 거래에 설정된 입찰 마감 기한을 기준으로 노출됨. 단,
  - 운영 정책상 조기 마감이 존재하는 경우에는 “확정 시
  - 각”과 “마감 기한”이 불일치할 수 있으므로, 본 화면에서
  - 는 마감 기한을 우선 노출 기준으로 고정함.
  - 참여 딜러 수는 입찰 레코드 중 상태=유효에 해당하는 데
  - 이터에서 딜러 기준 중복 제거(distinct) 후 집계된 값만
  - 포함됨. 취소/무효 상태 레코드는 집계 제외됨.
  - 최고 입찰가는 상태=유효 입찰 중 최대 금액 1건을 기준
  - 으로 산출됨. 동률이 발생하는 경우에는 “낙찰 규칙”에 의
  - 해 낙찰 딜러 단일 확정이 먼저 수행됨
  - 평균 입찰가는 상태=유효 입찰 금액들의 산술 평균으로
  - 산출됨. 소수점 처리 규칙은 플랫폼 공통 금액 규칙(원 단
  - 위 절사/반올림 등)을 그대로 적용하며, 산출 이후 값은
  - 재계산되지 않음.
  - 낙찰 딜러는 낙찰 규칙에 의해 단 1인으로 확정 노출됨.
  - 낙찰 확정 이후에는 딜러 변경 불가이며, 이후 검차/감가/
  - 인도/정산의 모든 주체 딜러 기준값으로 사용됨.
  - 본 요약 영역의 모든 값은 입찰 종료 확정 처리 완료 시점
  - 에 스냅샷으로 고정 저장된 값을 노출하며, 이후 입찰 내
  - 역의 상태 변경(취소 등)이 존재할 수 없도록 입찰 단계에
  - 서는 수정 경로가 차단됨.

### 2. 입찰 참여 내역(테이블)
  - 본 테이블은 거래에 생성된 “입찰 레코드”를 딜러 단위로
  - 집계한 목록이며, 입찰 종료 확정 이후에는 정렬/필터가
  - 바뀌더라도 원본 레코드가 변경되지 않는 조회 전용 상태
  - 로 유지됨.
  - 딜러명은 거래의 입찰 주체를 식별하는 기준값이며, 추후
  - 정산/분쟁/감사 시 “낙찰 딜러 확정 근거”로 사용됨. 딜러
  - 탈퇴/상태 변경이 발생해도 본 거래의 이력에서는 거래
  - 당시 명칭을 유지함(거래 이력 보존 우선).
  - 입찰가는 해당 딜러의 “유효 입찰 금액”으로 노출됨. “낙
  - 찰 판단에 사용된 최종 유효값”을 1건으로 고정해 노출하
  - 는 것을 원칙으로 함.
  - 입찰 시간은 낙찰 규칙 중 동률/우선순위 판단 근거가 될
  - 수 있는 로그값이며, 시간대 표기는 시스템 표준 시간대
  - 기준으로 고정됨.
  - 국가는 거래 유형(국내/해외) 및 후속 단계의 검차/정산
  - 정책 분기에 영향을 주는 분기값이며, 입찰 이후 변경 불
  - 가로 유지됨.
  - 상태는 각 딜러의 입찰 레코드가 낙찰 판단에서 어떤 결
  - 과로 확정되었는지 표현하며 다음의 제한을 가짐.
  - 낙찰는 낙찰 확정된 단 1건에만 부여됨(복수 존재 불
  - 가).
  - 패찰은 낙찰이 아닌 정상 입찰 레코드에만 부여됨.
  - 취소는 딜러 취소/무효 처리된 레코드이며, 상단 “참
  - 여 딜러 수/평균” 집계에서 제외됨.
  - 테이블 내 Highest 배지는 낙찰 확정 이후에도 유지되며,
  - 이후 단계로 넘어가더라도 낙찰 변경이 발생하지 않도록
  - 입찰 단계의 상태는 고정됨.
