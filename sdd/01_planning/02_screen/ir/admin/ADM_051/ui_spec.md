# ADM_051 UI Spec

- screen code: `ADM_051`
- screen name: `공지사항상세`
- primary route: `/admin/support/notices`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `52`

## Screen Requirements

### 1. 헤더/진입 정보
  - 현재 위치 ‘공지사항 관리’ 고정 표기 영역.
  - 좌측 뒤로가기 동작은 공지 목록 화면 복귀로 고정되며,
  - 복귀 시 목록의 조회 조건(검색/필터/페이지)은 진입 직
  - 전 상태로 유지

### 2. 공지 상세 본문 영역
  - 선택 공지의 확정 콘텐츠 표시 영역.
  - 제목: 목록의 ‘제목’ 컬럼과 동일 원본.
  - 메타 정보: 채널/카테고리/등록일(또는 최종 반영일) 노
  - 출.
  - 채널: 사용자 노출 대상(전체/판매자/딜러) 구분값.
  - 해당 값에 따라 사용자 앱/웹의 공지 목록 포함 여부
  - 결정됨.
  - 카테고리: 공지 분류값. 사용자 화면 내 카테고리 필
  - 터/탭이 존재할 경우 동일 기준으로 묶임.
  - 일자: 공지의 최종 수정 확정일
  - 본문: 서식 포함 텍스트(예: 번호 목록/강조/링크/첨부 아
  - 이콘 등) 그대로 렌더링되는 영역. 줄바꿈/목록/강조 스
  - 타일은 작성 화면에서 확정된 결과와 동일하게 유지됨.
  - 조회 범위: 본 화면은 열람 전용이며, 본문 직접 편집/임
  - 시 저장 개념 없음. 변경 필요 시 (5) 수정 진입을 통해서
  - 만 변경 가능 상태로 취급.

### 3. 목록으로
  - 공지 목록 화면 복귀 트리거.
  - 클릭 시 현재 공지 상세를 종료하고 목록으로 전환됨.
  - 목록 복귀 시 필터/검색/페이지 상태 유지 정책이 적용됨

### 4. 삭제
  - 공지 삭제 확정 진입 트리거.
  - 삭제는 “조회 화면에서 즉시 반영되는 운영 확정 행위”로
  - 취급되며, 삭제 완료 시 본 공지는 목록에서 제외됨
  - 사용자 노출 측면에서는 삭제 확정 시점부터 해당 채널의
  - 공지 목록/상세에서 접근 불가 상태로 전환됨.

### 5. 수정
  - 공지 수정 화면 진입 트리거.
  - 클릭 시 선택 공지의 기존 데이터를 유지한 채 수정 화면
  - (ADM_052)으로 전환됨.
  - 수정완료 시 목록 화면의 ‘수정일’ 갱신 기준일 확정
