# ADM_054 UI Spec

- screen code: `ADM_054`
- screen name: `정책등록상세`
- primary route: `/admin/policies/:policyId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `55`

## Screen Requirements

### 1. 정책 문서 입력 영역
  - 정책 문서의 분류·식별·내용·노출 여부를 확정하는 핵심
  - 입력 블록.
  - 카테고리
  - 정책 문서의 분류 값.
  - 사용자 화면의 정책/약관 분류 구조에 직접 사용되는
  - 기준 값.
  - 미선택 상태에서는 저장 불가
  - 버전(읽기 전용)
  - 문서 개정 이력 식별 값.
  - 신규 등록 진입 시 기본값(v0.1)로 시작.
  - 목록에서 “버전 보기”로 관리되는 이력 체계의 기준
  - 값으로 사용.
  - 수정(개정) 시 기존 버전 대비 증가된 버전 값이 표시
  - 됨
  - 문서 제목
  - 목록/사용자 화면에 노출되는 정책 문서명.
  - 문서 내용(서식 입력)
  - 정책 본문 입력 영역.
  - 서식(굵게/기울임/밑줄/목록/링크/이미지 등) 포함
  - 가능한 편집기 형태.
  - 사용자 화면에서는 입력된 서식이 유지된 형태로 렌
  - 더링되는 기준 데이터.
  - 빈 내용 저장 불가
  - 노출 여부(노출/비노출)
  - 사용자 서비스 화면 공개 여부를 확정하는 상태 값.
  - 노출: 사용자 화면에서 즉시 접근 가능 상태로 취급.
  - 비노출: 관리자 전용 문서로 취급(사용자 접근 불가).

### 2. 취소
  - 입력 내용 확정 없이 이탈하는 종료 트리거.
  - 저장 전 상태로 화면 종료.
  - 수정 진입 상태에서 취소 선택 시, 변경사항 폐기 후 이전
  - 화면으로 복귀

### 3. 저장
  - 정책 문서의 내용/노출 상태/버전 기준을 확정하는 최종
  - 실행 트리거.
  - 저장 성공 시 정책 문서 관리 목록의 해당 행 정보가 갱신
  - 됨(버전/노출여부/최종수정일/등록자).
  - 노출 상태가 “노출”인 경우 사용자 화면에 적용 가능한 기
  - 준 문서로 취급됨.
