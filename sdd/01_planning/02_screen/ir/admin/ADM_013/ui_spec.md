# ADM_013 UI Spec

- screen code: `ADM_013`
- screen name: `거래상세(감가)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `14`

## Screen Requirements

### 1. 딜러 감가 제안 요약
  - 제안 등록 시각
  - 딜러가 감가 제안을 최초 저장한 시각
  - 감가 협의의 기준 타임스탬프
  - 최초 낙찰가
  - 입찰 종료 시 확정된 원 거래가
  - 감가 산정의 기준 가격
  - 감가 총액
  - 감가 항목별 금액의 합계
  - 본 값은 이후 모든 정산·송금 계산의 기준 감액 값으
  - 로 고정
  - 최종 제안가
  - 최초 낙찰가 – 감가 총액
  - 본 거래의 최종 거래가
  - 인도·송금·정산 전 과정에서 변경 불가
  - 딜러 메모
  - 감가 사유 요약 설명
  - 판매자 승인 판단 근거로 사용된 기록 데이터
  - 상태 규칙
  - 판매자 승인 완료 시점부터 본 영역 데이터는 불변
  - 거래 취소를 제외한 모든 흐름에서 재산정 불가
  - 딜러 감가 상세보기 > 버튼 클릭 시 (ADM_028) 화면으
  - 로 이동

### 2. 감가 항목 상세
  - 각 감가 항목은 아래 구조로 확정 보존됨
  - 항목명
  - 감가 사유
  - 감가 금액
  - 합계
  - 감가 항목 총합
  - ①의 감가 총액과 항상 일치
  - 상태 규칙
  - 감가 항목 수정·삭제·추가 불가
  - 본 테이블은 최종 감가 계약서 성격의 기록 데이터

### 3. 판매자 응답 상태
  - 전달 시각 : 딜러 감가 제안이 판매자에게 전달된 시각
  - 응답 시각 : 판매자가 감가를 승인한 시각
  - 경과 시간 : 전달 시각 → 승인 시각까지의 실제 소요 시
  - 간
  - 상태 규칙
  - 승인 완료 이후 추가 응답·재협의 불가
  - 관리자·딜러 모두 본 영역 수정 불가

### 4. 감가 협의 로그
  - 감가 협의 전 과정의 시간순 이벤트 기록
  - 검차 리포트 전송
  - 딜러 감가 제안 등록
  - 판매자 감가 승인
  - 감가 확정 및 인도 단계 전환
  - 본 로그는 거래 종료 이후에도 삭제·수정 불가
  - 분쟁, 법적 검증, 정산 검증의 기준 데이터
