# ADM_033 UI Spec

- screen code: `ADM_033`
- screen name: `딜러송금관리`
- primary route: `/admin/remittances`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `34`

## Screen Requirements

### 1. 상단 필터/검색/갱신 영역
  - 딜러 송금 처리 대상을 운영 우선순위로 정렬하기 위한
  - 제어 영역.
  - 상태 필터는 전체 / 확인 대기 / 송금 완료 탭으로 구성되
  - 됨
  - 확인 대기는 딜러가 송금 후 “확인 요청” 이벤트를 발
  - 생시킨 상태로, 관리자 확정 처리 전 단계로 분류됨.
  - 송금 완료는 관리자가 입금 확인을 확정한 상태로, 이
  - 후 동일 거래에 대해 “딜러 송금 확정” 재처리 불가 상
  - 태로 유지됨(조회 전용).
  - 기간 필터는 오늘 / 최근 7일 / 최근 30일 / 전체로 구성
  - 되며, 기준은 송금일자 또는 확인요청 발생일시 중 운영
  - 정책값으로 고정됨(둘 중 무엇을 기준으로 집계하는지 서
  - 버 기준 확정 필요).
  - “전체” 선택 시 기간 조건 미적용 상태로 조회됨.
  - 검색 입력은 거래 ID / 차량번호 / 판매자 / 딜러 키워드
  - 복합 검색으로 동작하며, 단일 입력값을 서버에서 다중
  - 필드 OR 조건으로 매칭하는 방식으로 처리됨.
  - 초기화 버튼 클릭 시 상태/기간/검색 조건이 기본값으로
  - 리셋되고, 1페이지부터 재조회됨.
  - 갱신 아이콘 및 “기준 시각”은 현재 화면의 집계 스냅샷
  - 기준을 명시하는 정보로, 갱신 실행 시 기준 시각이 최신
  - 으로 갱신되어 재조회됨.

### 2. 딜러 송금 대상 목록 영역
  - 딜러 송금 단위를 거래 1건 단위로 나열하고, “확인 대기”
  - 처리 대상을 즉시 식별할 수 있도록 컬럼/상태 태그가 고
  - 정된 목록 영역.
  - 목록 상단의 “총 N건”은 현재 필터 조건에 의해 조회된 전
  - 체 결과 수로 고정 노출됨.
  - 테이블 컬럼 정의 및 데이터 기준
  - 거래ID: TR-YYYYMMDDNNN 형식의 거래 식별자.
  - 클릭 시 해당 거래의 송금 상세/확인 화면으로 이동되
  - 는 진입점으로 사용됨.
  - 차량 정보: 차량 모델/트림 등 대표 스펙 + 보조정보
  - (연식/주행거리 등)가 함께 노출되는 요약 값. 거래
  - 생성 시점의 차량 정보 스냅샷 기준으로 노출됨.
  - 판매자: 거래 판매자 계정의 표시명 노출
  - 딜러: 낙찰 확정된 딜러의 표시명 노출. 송금 주체 판
  - 단을 위한 핵심 값으로 고정됨.
  - 송금 금액: 딜러가 송금해야 하는 거래 금액. 통화 단
  - 위는 ₩ 고정이며, 금액은 거래의 “딜러 납부 금액” 확
  - 정값 기준으로 노출됨
  - 송금 일자: 딜러가 입력/제출한 송금일로 고정 노출
  - 됨.
  - 상태: 확인 대기 / 송금 완료 태그로 시각 구분되며, 목
  - 록 필터와 1:1로 대응되는 상태 값으로 관리됨.
  - SLA: D-n / D+n 형태의 경과 지표로 노출되며, 기준
  - 시점(예: 확인요청 발생 시각) 대비 현재까지 경과일
  - 을 산정한 운영 지표로 사용됨. 임계치 초과 시 강조
  - 컬러 정책 적용 가능.
  - 최종 업데이트: 해당 송금 건의 마지막 변경 시각(상
  - 태 변경, 메모 갱신, 확인 처리 등)을 의미하며, 운영자
  - 가 최신 처리 여부를 판단하는 근거로 사용됨.
  - 리스트행 클릭 시 송금상세(ADM_034) 화면으로 이동
