# ADM_056 UI Spec

- screen code: `ADM_056`
- screen name: `문의관리`
- primary route: `/admin/support/inquiries`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `57`

## Screen Requirements

### 1. 제목/이메일 검색
  - 입력값 기준 검색 범위: 문의 제목 + 문의자 이메일 동시
  - 대상.
  - 검색 실행 시 목록(3) 즉시 갱신, 필터(2) 조건과 결합 적
  - 용.
  - 검색어 공란 상태에서 실행 시 전체 결과 유지.
  - 초기화 버튼 클릭 시 검색어 제거 + 필터(2) 초기 상태로
  - 복귀 + 목록(3) 기본 정렬로 재노출.

### 2. 문의 필터 영역
  - 문의 채널
  - 문의 생성 주체 기준 분류.
  - 전체 / 판매자 / 딜러 3단 분기.
  - 선택 값은 목록(3)의 문의 채널 컬럼과 1:1 매칭.
  - 문의유형
  - 문의 등록 시 사용자가 선택한 문의 분류 값.
  - 드롭다운 단일 선택.
  - 전체 선택 시 유형 조건 미적용.
  - 처리 상태
  - 운영 처리 단계 분류.
  - 전체 / 문의 접수 / 답변완료 3단 분기.
  - 답변완료는 “답변 내용 확정 + 회신 발송 성공 + 처리
  - 일시 기록”을 만족한 건만 포함.
  - 기간
  - 문의 접수일 기준 필터.
  - 오늘 / 최근 7일 / 최근 30일 / 전체 탭 선택.
  - 기간 선택 시 목록(3)의 접수일 컬럼을 기준으로 결과
  - 제한.

### 3. 문의 목록 테이블
  - 목록은 필터(1)(2) 결과의 집합이며, 페이지네이션 단위
  - 로 탐색.
  - 기본 정렬 기준: 접수일 최신순(최신 문의 우선 노출).
  - 행 클릭 시 문의 상세 화면으로 이동(ADM_057).
  - No
  - 목록 내 표시용 순번.
  - 페이지/정렬 조건에 따라 변동 가능.
  - 문의 채널
  - 판매자 / 딜러 값 노출.
  - 문의 작성 주체 확정 기준.
  - 이메일
  - 문의자가 입력한 회신 수신 이메일.
  - 답변 발송 대상 주소로 사용되는 기준 값.
  - 문의 유형
  - 문의 분류 값 노출(예: 정산/송금, 감가협의, 계정/인
  - 증, 입찰/경매, 기타 등).
  - 상담 라우팅/운영 통계 기준으로 사용.
  - 제목
  - 문의 요약 제목.
  - 상태
  - 처리상태 태그로 시각 구분.
  - 접수 : 답변 미확정 상태.
  - 답변완료 : 답변 확정 및 회신 발송까지 완료된 상태.
  - 접수일
  - 문의가 시스템에 접수된 시각.
  - 기간 필터(2)의 기준 컬럼.
  - 담당자
  - 문의 처리 담당 운영자 식별자.
  - 접수 시점에 자동 지정
  - 처리일시
  - 답변완료로 확정된 시각.
  - 접수 상태에서는 -로 유지.
