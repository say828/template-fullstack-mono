- primary route: `/admin/audit/logs`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `63`

## Screen Requirements

### 1. 통합 검색(키워드)
  - 입력 대상: IP / 관리자명(또는 관리자ID) / 키워드
  - 키워드 매칭 범위
  - 수행 내역(상세) 문구 내 포함 검색
  - 대상 식별값(거래ID, 사용자ID, 딜러ID, 정책ID 등) 포
  - 함 검색
  - 접속 IP 정확/부분 일치 검색 지원(예: 192.168 단위)
  - 검색 실행 기준
  - 엔터 또는 검색 아이콘 기준 검색 확정
  - 검색 조건 확정 시 목록/총 건수 동기 갱신

### 2. 필터 영역(로그 유형 / 결과 / 기간)
  - 로그 유형
  - 전체/개별 유형 선택 기준으로 목록 필터링
  - 유형 예시 범주: 거래관리, 검차/감가, 인도관리, 정산
  - 관리, 사용자관리, 콘텐츠관리, 시스템설정 등
  - 결과
  - 전체/성공/실패 필터 제공
  - 실패 포함 시 운영 조치 필요 건(오류/권한 부족/유효
  - 성 실패 등) 확인 목적
  - 기간
  - 오늘/최근 7일/최근 30일/전체 프리셋 제공
  - 기간 변경 시 목록/총 건수 즉시 재집계 반영
  - 필터 조합 규칙
  - (로그 유형 × 결과 × 기간 × 키워드) 교집합 결과만 노
  - 출
  - 선택값이 “전체”인 항목은 해당 조건 미적용 상태로
  - 간주

### 3. CSV 다운로드
  - 다운로드 범위
  - 현재 화면에 적용된 검색 조건(1,2번 포함) 결과 전체
  - 를 대상으로 추출
  - 페이지에 노출된 일부 행만이 아니라 조건 결과 전체
  - 가 추출 대상
  - 화면 목록 컬럼(4번)과 동일 항목 + 추적을 위한 식별
  - 값(필요 시) 포함
  - 다운로드 권한 보유자만 노출 또는 실행 가능
  - 다운로드 실행 자체도 로그로 표시

### 4. 로그 목록(감사 테이블)
  - 목록 목적: 행위 단위 감사 증빙 1행 1기록
  - 발생 일시: 행위가 확정된 시각(서버 기준 시각 일관
  - 적용)
  - 관리자: 수행 주체(관리자명 + 관리자ID 표기)
  - 접속 IP: 행위 수행 시점의 접속 원천 IP
  - 로그 유형: 행위가 속한 업무 범주(메뉴/업무 구분)
  - 수행 내역(상세): 실제 수행한 행동이 문장 단위로 요
  - 약된 기록(승인/수정/다운로드/상태 변경 등), 핵심
  - 파라미터 포함
  - 대상: 행위가 영향을 준 1차 대상 식별값(거래ID/사용
  - 자ID/딜러ID/정책ID 등)
  - 결과: 성공/실패 상태 태그로 구분 노출
  - 최신 발생 일시 기준 내림차순 기본
