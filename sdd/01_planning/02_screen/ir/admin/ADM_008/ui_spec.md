# ADM_008 UI Spec

- screen code: `ADM_008`
- screen name: `거래관리요약`
- primary route: `/admin/trades/summary`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `9`

## Screen Requirements

### 1. 거래 기본 정보 영역
  - 거래 고유 식별값(TR-20250220) 표시.
  - 차량 기본 정보 표시 (연식 / 모델).
  - 현재 거래의 상위 단계 상태 태그 표시: 감가협의.
  - 해당 정보는 거래 테이블 기준값과 실시간 동기화됨

### 2. 다음 필요 작업 알림
  - 해당 거래의 현재상태(예: 감가협의) 내에서, 지금 시점에
  - 거래가 진행되기 위해 다음으로 완료되어야할 세부 단계
  - 를 표시하는 영역
  - 거래가 다음으로 어떤 절차를 기다리고 있는지를 직관적
  - 으로 전달하기 위한 가이드 영역
  - 해당 알림은 거래 단계 상태가 변경되는 즉시 자동 갱신
  - 됨.

### 3. 담당 관리자
  - 현재 거래의 관리 책임자로 지정된 관리자 계정 표시.
  - 관리자가 배정되지 않은 경우 “미배정” 상태 표시됨.
  - 배정 정보는 거래 단위로 고정 관리됨.

### 4. 최근 이슈
  - 해당 거래의 운영 중 발생한 지연·이탈·리스크 상태 요약
  - 표시.
  - “감가 협의 지연 (판매자 승인 대기)”
  - → 감가 협의 완료 SLA를 초과했으며 판매자 승인 미도
  - 착 상태임을 의미.
  - 이슈 상태는 SLA 계산 로직과 연동됨.
  - 이슈가 없을 경우, 해당 영역은 비노출됨

### 5. 거래 타임라인
  - 해당 거래의 주요 단계 이벤트 이력 시간순 기록.
  - 이벤트 유형
  - 발생 일시
  - 처리 주체
  - 최종 행은 현재 거래의 확정 단계 상태를 표시

### 6. 거래 상세 보기
  - 클릭 시 해당 거래의 현재 단계 기준 운영 처리 화면
  - (ADM_009 등)으로 이동.
  - 이동 시 본 요약 패널 종료 및 메인 콘텐츠 영역 전환됨.
