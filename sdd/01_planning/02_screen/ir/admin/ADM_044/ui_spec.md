# ADM_044 UI Spec

- screen code: `ADM_044`
- screen name: `반려처리`
- primary route: `/admin/dealers/:dealerId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `45`

## Screen Requirements

### 1. 반려 사유 입력 영역
  - 반려 확정의 선행 조건을 충족시키는 핵심 입력 영역.
  - 반려 처리 안내 배너는 “반려 사유 입력이 필수”임을 고지
  - 하며, 작성 내용이 딜러에게 그대로 전달되는 성격임을
  - 명확히 노출.
  - 입력값은 반려 확정 시점에만 확정 저장되며, 확정 이전
  - 에는 임시 입력 상태로 취급.
  - 줄바꿈/서식(굵게/기울임/밑줄/목록)/링크/이미지 삽입
  - 등 편집 기능이 제공되는 구성(툴바 노출 기준)

### 2. 취소 버튼
  - 반려 확정을 취소하고, 딜러 상세 패널(승인대기)로 복귀
  - 시키는 종료 트리거.
  - 클릭 시 입력된 반려 사유는 확정되지 않은 상태로 취급
  - 되며, 저장 없이 팝업 종료

### 3. 반려 확정 버튼
  - 반려 사유를 확정 저장하고 딜러 반려 처리를 완료시키는
  - 최종 트리거.
  - 클릭 시 1번 입력값 필수 충족 여부 및 메일 발송 대상(딜
  - 러 이메일)의 존재 여부 검증
  - 확정 처리 시
  - 딜러 상태: 승인대기 → 반려 전환
  - 반려 사유: 반려 이력에 확정 값으로 저장
  - 반려 안내 메일: 반려 사유 포함하여 딜러 이메일로
  - 발송
  - 화면 반영: 팝업 닫힘과 동시에 딜러 상세 패널/목록
  - 의 상태 및 노출 버튼이 반려 기준으로 갱신
