## Screen Requirements

### 1. 판매자 조율 요청 정보 영역
  - 본 영역은 거래 상태가 검차 일정 조율 요청인 경우에만
  - 노출됨
  - 판매자가 최초 제안 일정에 대해 수정 요청한 원본 요청
  - 데이터 스냅샷 표시
  - 운영자는 본 영역 데이터를 수정 불가, 판단 기준값으로
  - 만 사용
  - 요청 일시: 판매자 조율 요청 제출 시각
  - 희망 날짜 : 판매자 요청 검차 날짜
  - 희망 시간대 : 판매자 요청 검차 시간 범위
  - 선호 지역 : 판매자 요청 검차 지역
  - 특이사항 : 판매자 입력 원문 요청 사유
  - 해당 데이터는 이후 일정 확정 시 조율 로그 기준 데이터
  - 로 고정 저장됨

### 2. 검차 일정 재확정 입력 영역
  - 판매자 조율 요청을 반영하여 최종 검차 일정 확정값을
  - 입력하는 제어 영역
  - 이 영역에서 확정된 값이 검차 단계 전체 진행 기준 데이
  - 터가 됨
  - 검차 일시 : 검차 확정 일시
  - 담당 평가사 배정 : 검차 수행 평가사 단일 확정
  - 검차 장소 : 검차 센터 단일 확정
  - 평가사 연락처 : 선택된 평가사 정보 자동 바인딩
  - 모든 필수 항목 입력 완료 상태에서만 전송 가능
  - 전송 성공 시 거래 상태 → 검차 진행 가능 상태 전환
  - 판매자가 검차일정 승인 시 검차 결과 영역 활성화
