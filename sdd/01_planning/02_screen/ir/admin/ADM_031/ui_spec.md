# ADM_031 UI Spec

- screen code: `ADM_031`
- screen name: `인도관리상세(승인대기)`
- primary route: `/admin/deliveries/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `32`

## Screen Requirements

### 1. 인도 일정 정보 영역
  - 해당 거래의 인도 일정 레코드 확정 상태 표시 영역.
  - 인도 일시, 인도 장소 값은 인도 일정 테이블에 저장된 확
  - 정 데이터 기준 고정 노출

### 2. 인도 완료 확인 영역
  - 판매자 확인, 딜러 확인 모두 완료 상태로 고정 노출
  - 각 확인 값은 개별 주체의 인도 완료 확인 이벤트 발생 시
  - 점에 생성된 로그 기반으로 표시
  - 양측 확인이 모두 완료되기 전까지 5번 인도 완료 승인
  - 버튼 활성 불가 상태 유지.

### 3. 차량 인도 사진 영역
  - 딜러가 업로드한 차량 인도 증빙 이미지 목록 노출 영역.
  - 파일은 인도 완료 이벤트와 함께 인도 증빙 테이블에 귀
  - 속 저장되며, 승인 처리 시 기준 증빙 자료로 활용됨
  - 승인 이후 변경·삭제 불가 상태로 고정 처리.

### 4. 서류 인수 확인서 영역
  - 차량 인수 서류 PDF 파일 고정 노출 영역
  - 본 파일은 인도 완료 확인 단계에서 업로드된 단일 확정
  - 증빙 문서이며, 승인 처리 이후 수정 불가 상태 유지.

### 5. 인도 완료 승인 버튼
  - 관리자의 최종 승인 트리거 영역.
  - 활성 조건은 다음 세 항목 동시 충족 상태일 때만 성립:
  - (인도 일정 레코드 존재/ 판매자 인도 완료 확인 = 완료 /
  - 딜러 인도 완료 확인 = 완료)
  - 버튼 클릭 시 거래 상태가 ‘인도 완료’ 단계로 전환되며,
  - 이후 송금·정산 프로세스가 자동 개시됨.
