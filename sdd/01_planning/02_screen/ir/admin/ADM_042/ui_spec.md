## Screen Requirements

### 1. 조회 조건 영역(상태/기간/검색/초기화)
  - 딜러 목록을 운영 목적에 맞게 좁혀서 조회하기 위한 조
  - 건 입력 영역.
  - 상태 탭
  - 선택값: 전체 / 승인대기 / 반려 / 정상 / 블랙리스트 /
  - 탈퇴
  - 상태는 딜러 계정의 운영 가능 여부 및 제재 여부를
  - 대표하는 기준값이며, 목록/상세에서 동일 값으로 일
  - 관 노출
  - 탭 선택 시 목록은 해당 상태로 즉시 재구성되며, 페이
  - 지는 1페이지로 리셋
  - “전체”는 모든 상태를 포함하되, 정렬은 가입일 기준
  - 기간 필터
  - 옵션: 오늘 / 최근 7일 / 최근 30일 / 전체
  - 기간의 기준 필드는 화면 상단 “기준 시각” 및 가입일
  - 로 고정.
  - “전체”는 기간 필터 미적용 상태로 취급.
  - 검색
  - 입력 플레이스홀더 기준: 딜러ID / 딜러명 / 업체명 /
  - 연락처 검색
  - 다중 키워드가 아닌 단일 키워드 검색 기준으로 처리.
  - 초기화
  - 클릭 시 상태=전체, 기간=전체, 검색어=빈값으로 복
  - 구.
  - 초기화 후 목록은 “초기 기본 정렬/기본 페이지(1페
  - 이지)”로 복원.
  - 기준 시각(상단 표기)
  - 화면에 노출되는 집계/목록이 어느 시점 기준인지 명
  - 시하는 운영 안전장치.
  - 기준 시각은 “화면 재진입/새로고침” 시 갱신

### 2. 딜러 목록 영역(집계/테이블/페이지네이션/행 선택)
  - 조건(1번)에 의해 산출된 딜러 후보를 목록으로 제공하
  - 며, 행 선택을 통해 상세 패널 진입을 확정하는 영역.
  - 총 건수
  - 현재 조건(상태/기간/검색)이 적용된 결과의 총합을
  - 의미.
  - 페이지네이션과 무관한 전체 결과 수로 고정(현재 페
  - 이지 행 수가 아님)
  - 딜러ID: 딜러 가입/등록 시 부여된 고유 식별자.
  - 딜러명: 딜러 개인 또는 대표자 명칭. 동일 이름 중복
  - 가능성이 있으므로 운영 판단 기준은 ID 우선.
  - 업체명: 딜러가 소속/운영하는 사업체명.
  - 활동국가: 딜러 활동 기준 국가. 국가별 운영 정책(서
  - 류/세금/정산)이 존재하는 경우 분기 판단 근거로 사
  - 용.
  - 이메일 / 연락처: 운영 커뮤니케이션 수단. 연락처는
  - 표시 포맷을 고정(국가코드 포함 여부 등)하여 검색
  - 일관성 유지 필요.
  - 입찰 횟수: 딜러가 실제로 입찰에 참여한 누적 횟수
  - 누적 매입: 딜러가 매입 확정한 누적 건수
  - 상태: 운영 상태값을 컬러 태그로 시각 구분(승인대
  - 기/반려/정상/블랙리스트/탈퇴). 탭(1번)과 동일한
  - 원천값으로 동기화되는 전제.
  - 가입일: 딜러 계정 생성/승인 요청 시작 시점
  - 상태가 승인대기 인 경우의 리스트 행 클릭 시
  - (ADM_043) 화면을 호출하며, 그 외 상태값의 경우
  - 딜러상세(ADM_045) 화면을 호출함
