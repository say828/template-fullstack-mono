# ADM_007 UI Spec

- screen code: `ADM_007`
- screen name: `거래관리`
- primary route: `/admin/trades`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `8`

## Screen Requirements

### 1. 기준 시각
  - 본 화면의 전체 데이터 산출 기준 시각 표시됨.
  - 모든 필터·집계·SLA 계산은 해당 시각 기준으로 처리됨.
  - 새로고침 시 기준 시각 갱신 및 테이블 데이터 재조회 수
  - 행됨.

### 2. 상태/기간/검색 필터 영역
  - 거래 목록의 조회 범위를 제한·조합하는 제어 영역.
  - 모든 필터 조건은 동시 적용되며, 조건 변경 즉시 서버 재
  - 조회 수행됨.
  - 전체: 모든 상태 포함
  - 입찰중: 낙찰 이전 단계의 거래
  - 검차: 검차 진행 단계에 존재하는 거래
  - 감가협의: 감가 협의 단계에 존재하는 거래
  - 인도/정산: 인도 이후 송금·정산 단계에 존재하는 거래
  - 거래완료: 정산까지 종료된 거래
  - 취소: 거래 중단·취소 확정 거래
  - 기간 필터 : 오늘 / 최근 7일 / 최근 30일 / 전체
  - → 거래 생성일 기준 적용
  - 검색 : 거래ID / 차량번호 / 판매자 / 딜러 통합 검색 (부분
  - 일치 허용)

### 3. 거래 집계 요약
  - 현재 필터 조건이 적용된 거래의 운영 요약 지표 표시됨.
  - 총 거래: 전체 거래 건수
  - 위험 거래: SLA 초과 또는 문제 상태로 분류된 거래 건수
  - 오늘 신규: 기준 시각 당일 생성된 거래 건수

### 4. 거래 목록 테이블
  - 각 행은 1개의 등록된 거래 단위이며, 거래의 현재 운영
  - 상태를 결정하는 모든 핵심 지표를 포함함.
  - 거래ID: 거래 고유 식별값
  - 차량정보: 차량 연식 / 모델 / 주행거리 요약.
  - 판매자: 판매자명.
  - 딜러: 낙찰 딜러명.(입찰중인경우, ‘-’ 표시)
  - 상태: 거래의 현재 상위 단계 상태 태그 표시.
  - 검차 / 감가 / 인도 / 송금 / 정산: 각 단계별 진행 여부 표
  - 시 아이콘. (● = 완료 / ○ = 미완료 / 색상으로 단계 상태
  - 구분됨.)
  - SLA: 현재 단계 기준 SLA 경과 일수 표시.
  - - D+N = 기준 시간 초과 N일
  - - D-N = 잔여 N일
  - 최종 업데이트: 해당 거래의 마지막 상태 변경 시각.
  - 클릭 시 해당 거래의 요약(ADM_008) 패널이 호출됨

### 5. 페이지당 노출 개수 설정
  - 페이지당 게임 노출 개수 선택
  - 선택 값 변경 시 즉시 목록 재조회
  - 기본값 10개

### 6. 페이지네이션
  - 전체 게임 수 기준 페이지 이동 제공
  - 페이지 이동 시 현재 필터·검색 조건 유지
  - 마지막 페이지 기준 데이터 부족 시 빈 행 미노출
