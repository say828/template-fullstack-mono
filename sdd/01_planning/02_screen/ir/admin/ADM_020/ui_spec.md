# ADM_020 UI Spec

- screen code: `ADM_020`
- screen name: `검차운영상세(제안필요)`
- primary route: `/admin/inspections/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `21`

## Screen Requirements

### 1. 헤더(거래 식별)
  - 화면 컨텍스트는 거래ID(TR-20250220) 기준으로 고정
  - 됨.
  - 본 화면에서 발생하는 모든 저장/전송/로그는 해당 거래
  - ID에 귀속됨.
  - 좌측 상단 뒤로가기 클릭 시 검차 운영 목록으로 이동됨
  - (목록의 필터/페이지 유지 정책은 목록 화면 정책에 종속
  - 됨).

### 2. 거래 화면 보기(새 창)
  - 클릭 시 해당 거래의 프론트 거래 상세 URL 새 창 오픈됨.
  - 오픈 대상은 관리자 화면이 아닌 딜러 노출 페이지이며,
  - 운영자가 사용자 관점으로 상태·문구·노출 데이터를 교차
  - 검증하는 목적임.
  - 본 화면 데이터/상태 변경 없음(조회 액션 전용).

### 3. 거래 상세 정보
  - 검차 운영 판단의 기준 데이터로 고정 노출됨
  - 포함 데이터:
  - 차량 정보: 연식/모델명, 주행거리(표시용), 차량 기본
  - 식별정보
  - 거래 관계자: 판매자, 딜러(낙찰 딜러 기준)
  - 현재 상태: 거래의 상위 단계 상태 태그(예: 검차요청/
  - 검차중/검차완료 등, 서비스 상태 정의에 종속)
  - SLA: 현재 단계 기준 잔여/초과 지표(D-0 등)
  - 담당 관리자: 운영 담당자 식별값(이름/ID)
  - 우측 상태·SLA는 본 화면의 일정 제안 전송/결과 확정 시
  - 점에 따라 자동 갱신 대상으로 취급됨.

### 4. 검차 일정 제안 입력 영역
  - 입력 완료 후 5번 전송을 통해 검차 일정 확정(제안 전송
  - 완료) 상태로 전환되는 선행 조건 영역.
  - 검차 일시: YYYY-MM-DD + HH + MM 조합으로 저
  - 장됨(시간/분은 드롭다운)
  - 담당 평가사 배정: 평가사 풀에서 1명 단일 선택(선택
  - 값은 평가사 식별자 기반 저장)
  - 검차 장소: 센터 목록 단일 선택(센터 식별자 기반 저
  - 장)
  - 평가사 연락처: 배정된 평가사 선택 시 자동 바인딩
  - 값(직접 입력 불가/읽기 전용)
  - 본 영역 값은 전송 전까지 임시 입력 상태이며, 전송 성공
  - 시 검차 일정 레코드로 확정 저장됨.

### 5. 검차 일정 제안 전송(확정 트리거)
  - 클릭 시 4번 입력값 유효성 검증 후 저장 시도됨.
  - 저장 성공 시:
  - 거래 상태가 ‘응답대기’ 로 전환됨
  - 검차 일정 레코드가 확정 생성됨(일시/평가사/센터
  - 포함)
  - 관련 알림 발송 트리거 발생(판매자/딜러/평가사 대
  - 상, 채널은 알림 정책에 종속)
  - 검차 결과 영역(6)의 상태가 ‘일정 미확정 안내’ → ‘결
  - 과 입력 가능 대기’로 전환됨

### 6. 검차 결과 영역(상태 기반 노출)
  - 검차 일정이 확정되지 않은 경우:
  - “검차 일정이 아직 확정되지 않았습니다” 안내 문구
  - 노출
  - 결과 입력 UI/첨부 UI 미노출 또는 비활성 유지
  - 검차 일정 확정 이후:
  - 검차 결과 입력/표시 컴포넌트 활성 전환 대상
  - 검차 결과 확정 시 감가 단계에서 참조되는 판단 근거
  - 데이터로 고정 취급됨
