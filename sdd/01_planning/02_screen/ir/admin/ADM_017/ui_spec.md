# ADM_017 UI Spec

- screen code: `ADM_017`
- screen name: `관리자강제종료`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `18`

## Screen Requirements

### 1. 거래 정보 요약 영역
  - 해당 팝업이 제어하는 거래 대상을 명확히 고정 표시
  - 거래ID
  - 차량 정보
  - 판매자 / 딜러 식별 정보
  - 현재 거래 단계 (입찰·검차·감가·인도·송금·정산 중 하나)
  - 하단 경고 문구 고정 노출

### 2. 강제 종료 사유 선택
  - 관리자가 종료 사유를 반드시 선택해야 다음 단계 진행
  - 가능
  - 사유 선택 없이는 종료 처리 불가

### 3. 상세 사유 입력
  - 최소 50자 이상 필수 입력
  - 종료 결정의 정당성·근거·사후 검증용 기록
  - 이 내용은 상태 이력 및 감사 로그에 영구 저장

### 4. 최종 보안 확인
  - 관리자 실수 방지를 위한 이중 인증 단계
  - 관리자 비밀번호 재확인
  - "강제 종료" 문구 직접 입력 확인
  - 모든 조건 충족 시에만 최종 실행 가능
  - 해당 구간 미완성 시 종료 버튼 비활성화

### 5. 취소 버튼
  - 모든 입력 초기화
  - 팝업 닫힘
  - 거래 상태 변경 없음

### 6. 강제 종료 실행
  - 모든 검증 조건 충족 시 활성화
  - 클릭 시 즉시 다음 처리 실행
  - 거래 상태 → ‘취소’로 변경
  - 모든 후속 단계 접근 차단
  - 모든 액션 버튼 비활성화
  - 거래 상태 이력에 강제 종료 기록 생성
  - 감사 로그·관리자 ID·사유·시간 영구 저장
