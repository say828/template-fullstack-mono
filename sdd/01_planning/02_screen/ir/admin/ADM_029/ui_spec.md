# ADM_029 UI Spec

- screen code: `ADM_029`
- screen name: `인도관리`
- primary route: `/admin/deliveries`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `30`

## Screen Requirements

### 1. 인도 필터 / 조회 조건 영역
  - 본 영역은 인도 프로세스의 상태 단계를 기준으로 거래
  - 집합을 선별하는 필터영역.
  - 일정 상태
  - • 일정 대기: 인도 일정 미등록 상태
  - • 일정 확정: 인도 일정이 등록·확정된 상태
  - 인도 상태
  - • 인도 대기: 일정은 확정되었으나 실제 인도 미완료
  - • 승인 대기: 딜러·판매자 양측 인도 완료 확인이 제출되
  - 어 관리자 승인만 남은 상태
  - • 인도 완료: 관리자 승인까지 완료되어 인도 프로세스
  - 종결 상태
  - 기간 필터는 거래의 최종 업데이트 시각 기준으로 조회
  - 범위를 제한함.
  - 검색 필드는 거래ID / 차량번호 / 판매자 / 딜러 기준 부
  - 분 일치 조회 수행.
  - 모든 필터 조건 변경은 즉시 서버 조회 기준으로 반영됨.

### 2. 인도 상태 집계 영역
  - 본 영역의 수치는 현재 필터 조건이 적용된 거래 집합 기
  - 준 실시간 집계값임.
  - • 각 수치는 인도 프로세스의 병목 지점을 운영자가 즉시
  - 파악하기 위한 운영 KPI로 사용됨.
  - 집계 기준:
  - • 일정 대기: 인도 일정 레코드 미존재 거래 수
  - • 인도 대기: 일정 확정 후, 어느 한쪽이라도 인도 완료 미
  - 확인 상태 거래 수
  - • 승인 대기: 딜러·판매자 인도 완료 확인 모두 존재 + 관
  - 리자 승인 미완료 거래 수
  - • 인도 완료: 관리자 승인 완료 상태 거래 수
  - • 지연: SLA 기준 초과 거래 수

### 3. 인도 거래 리스트
  - 각 행은 1건의 거래(거래ID 기준)이며 인도 프로세스의
  - 단일 관리 단위임.
  - 리스트행 클릭 시 해당하는 거래의 인도관리상세
  - (ADM_030 등) 화면으로 이동
  - 거래ID
  - • 거래 생성 시 서버 발급 고유 키
  - 차량 정보
  - • 차종 / 연식 / 주행거리 요약 데이터
  - • 차량 식별 및 현장 인도 확인 기준 정보
  - 판매자 / 딜러
  - • 인도 책임 주체 식별 기준 사용자
  - 인도 일정
  - • 인도 일정 확정 시점의 날짜·시각
  - • 일정 미확정 시 공란 유지
  - 일정 상태
  - • 일정 대기 / 일정 확정 상태값
  - • 인도 프로세스의 선행 조건 여부 판단 기준
  - 인도 상태
  - • 인도 대기 / 승인 대기 / 인도 완료 상태값
  - • 인도 프로세스의 현재 단계 식별 핵심 필드
  - 증빙 제출
  - • 딜러·판매자 인도 증빙 제출 여부 상태
  - • 승인 가능 여부 판단 선행 조건
  - SLA
  - • 인도 기준 시한 대비 초과 여부
  - • 지연 판단 및 운영 리스크 지표
  - 최종 업데이트
  - • 인도 관련 이벤트 발생 최종 시각
  - • 정렬·운영 판단 기준 타임스탬프
