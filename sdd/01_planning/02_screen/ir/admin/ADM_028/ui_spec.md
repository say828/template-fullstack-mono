# ADM_028 UI Spec

- screen code: `ADM_028`
- screen name: `감가협의상세(재협의)`
- primary route: `/admin/depreciations/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `29`

## Screen Requirements

### 1. 감가 제안 이력
  - 본 영역은 거래 단위 감가 협의에서 딜러가 판매자에게
  - 전달한 감가 제안 스냅샷을 고정 출력하는 영역이며, “재
  - 협의” 상태의 기준점(직전 제안)으로 사용됨.
  - • 상단 금액 3종은 동일 이력 레코드에서 파생되는 값으
  - 로 고정 계산됨.
  - 원 낙찰가: 입찰 종료 시점에 확정된 낙찰 금액(낙찰 확정
  - 레코드 기준)
  - 제안 감가 금액: 딜러가 제안한 총 감가액(항목 합계와 정
  - 합성 필수)
  - 제안 후 최종 거래가(예상): 원 낙찰가에서 제안 감가 금
  - 액을 차감한 값(표시 전용, 협의 종료 시 “최종 거래가”로
  - 확정 전환됨)
  - • “감가 사유”는 항목 테이블의 상세 근거와 분리된 요약
  - 텍스트이며, 운영상 분쟁 시에는 항목 테이블이 1차 근거,
  - 감가 사유 요약은 2차 보조 근거로 취급됨.
  - • 하단 항목 테이블은 감가 산정의 근거 데이터로 고정되
  - 며, 각 행은 “항목명/내용/감가액” 3요소를 필수로 포함
  - 함.
  - 항목명: 서비스에서 정의한 감가 분류 키(예: 외관/타이
  - 어/실내 등)
  - 내용: 딜러가 감가 사유를 구체화한 설명
  - 감가액: 해당 항목에만 귀속되는 차감 금액(음수 표기 기
  - 준 고정)
  - • 합계 행은 항목별 감가액의 산술 합으로 산출되며, 합
  - 계 ≠ 제안 감가 금액인 경우 “비정상 감가 제안”으로 판단
  - 되어 재협의/확정 처리 기준 데이터로 사용 불가 상태로
  - 분기됨.
  - • 본 이력은 “제안 전송 시점”에 확정 저장된 레코드이며,
  - 화면 내에서 수정 불가.

### 2. 재협의 요청 내역
  - 본 영역은 1번 감가 제안에 대해 판매자 또는 딜러가 재
  - 협의 요청을 생성한 응답 레코드를 누적 노출하는 영역이
  - 며, “재협의” 상태의 직접 원인 데이터로 사용됨.
  - 재협의 블록은 요청 주체/요청 시각/요청 메시지/첨부 증
  - 빙을 한 묶음으로 고정 포함하며, 동일 거래 내 재협의가
  - 반복될 경우 블록이 누적됨.
  - “판매자 재협의 요청” 문구와 본문은 판매자가 입력한 원
  - 문을 보존하는 값이며, 운영 개입 시에도 원문 변경 불가
  - 가 전제되어야 로그/분쟁 근거로 성립함.
  - 우측/하단의 금액 표시는 재협의 단계에서 “판매자가 원
  - 하는 목표 금액” 또는 “판매자 주장 기준 거래가”로 해석
  - 되는 보조 값이며, 시스템 정책상 이 값은 “확정 금액”이
  - 아니라 재협의 판단을 위한 주장 데이터로 저장됨(최종
  - 확정은 별도 승인 이벤트에서만 발생).
  - 첨부 파일은 재협의 주장에 대한 증빙으로, 파일 목록은
  - 서버 저장 원본을 링크로 노출하며, 다운로드는 가능하나
  - 수정/삭제는 불가 상태로 유지됨
  - 본 영역의 최신 블록이 존재하는 동안 거래 상태는 “재협
  - 의”로 유지되며, 재협의가 해소되어 판매자 승인 확정 이
  - 벤트가 발생하면 “최종 확정 이력”이 생성되고, 그 시점부
  - 터 본 영역은 조회만 가능(신규 블록 생성 불가)
