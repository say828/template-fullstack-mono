# ADM_053 UI Spec

- screen code: `ADM_053`
- screen name: `정책문서관리`
- primary route: `/admin/policies`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `54`

## Screen Requirements

### 1. 검색
  - 입력한 키워드를 기준으로 정책 문서 목록이 조회됨
  - 검색 기준: 문서 제목
  - 검색 결과가 없을 경우에도 검색어는 유지된 상태로 목록
  - 이 노출됨
  - 검색 조건이 적용된 상태에서 필터 변경 시에도 검색 조
  - 건은 유지됨

### 2. 필터
  - 카테고리
  - 전체 / 서비스 이용약관 / 개인정보 처리방침 / 커뮤
  - 니티 가이드라인 / 운영정책 등 선택 가능
  - 정책 문서의 유형 기준으로 목록이 필터링됨
  - 노출여부
  - 전체 / 노출 / 비노출 선택 가능
  - 실제 서비스 화면에 적용 중인 정책 문서 여부 기준으
  - 로 구분됨

### 3. 정책 등록 버튼
  - 클릭 시 정책 문서 등록 화면(ADM_054)을 호출함
  - 신규 정책 문서 또는 신규 버전 정책을 등록할 수 있음
  - 등록 완료 후 정책 문서 목록 화면으로 복귀됨

### 4. 정책 문서 목록 테이블
  - 등록된 정책 문서가 리스트 형태로 노출됨
  - No : 정책 문서 관리용 순번 값
  - 문서 제목
  - 정책 문서의 대표 제목
  - 카테고리 : 정책 문서 유형 분류 값
  - 버전
  - 정책 문서의 현재 버전 정보
  - 동일 문서의 개정 이력을 구분하기 위한 기준
  - 노출여부
  - 정책 문서의 현재 적용 상태
  - 노출 상태인 경우 서비스 화면에 적용 중임을 의미함
  - 등록자 : 해당 정책 문서를 등록한 관리자 계정
  - 최종 수정일 : 정책 문서가 마지막으로 수정된 일자
  - 버전 보기 : 클릭 시 해당 정책 문서의 버전 이력 목록을
  - 확인할 수 있는 화면(ADM_055)을 호출함
  - 리스트행 클릭 시 해당하는 문서의 상세화면(ADM_054)
  - 을 호출함
