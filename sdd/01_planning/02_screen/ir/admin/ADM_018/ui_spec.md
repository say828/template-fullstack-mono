# ADM_018 UI Spec

- screen code: `ADM_018`
- screen name: `거래상세(취소)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `19`

## Screen Requirements

### 1. 강제 종료 상태 안내 영역
  - 본 거래가 관리자에 의해 강제 종료되었음을 시스템 상단
  - 에서 고정 노출로 선언하는 영역
  - 해당 배너 노출 시점부터 본 거래는 다음 제약 상태로 전
  - 환됨
  - 모든 거래 단계 접근 영구 차단
  - 검차·감가·인도·송금·정산 관련 기능 비활성화
  - 거래 상태, 이력, 금액 정보 수정 불가
  - 모든 사용자 액션 버튼 비노출 또는 비활성화
  - 배너 우측에는 종료 확정 시각을 고정 노출하여, 해당 거
  - 래가 언제 비가역 상태로 확정되었는지 명확히 기록됨

### 2. 종료 사유 요약 영역
  - 강제 종료 당시 관리자가 입력한 종료 사유 및 판단 근거
  - 요약을 표시
  - 종료 사유 유형 (분쟁 해결 실패, 법적 요청 등)
  - 종료 근거 상세 내용
  - 종료 처리자 (관리자 계정)
  - 이 영역은 감사·법무 검증을 위한 증빙 고정 데이터 영역
  - 으로 어떠한 경우에도 수정·삭제 불가 상태로 유지됨.

### 3. 거래 상세 정보 고정 영역
  - 강제 종료 이후에도 거래 식별을 위해 다음 정보는 조회
  - 전용으로 고정 유지.
  - 차량 정보
  - 거래 관계자(판매자/딜러)
  - 종료 당시 거래 단계
  - SLA
  - 담당 관리자
  - 거래 최종 상태: 취소
  - 분쟁 상태: 관리자 종료
  - 본 영역의 모든 데이터는 락(lock) 상태로 유지되며 추가
  - 변경·정정·보정 불가.

### 4. 상태 이력 탭 (강제 종료 로그 포함)
  - 강제 종료 이전의 모든 거래 진행 기록과 강제 종료 행위
  - 자체가 하나의 최종 이력으로 추가됨.
  - 상태 이력에는 다음 정보가 반드시 포함됨 ( 강제 종료 실
  - 행 시각, 실행 관리자 계정, 종료 사유 요약)
  - 이 로그는 플랫폼 최종 기록으로 취급되며 삭제·수정·마
  - 스킹 불가.
