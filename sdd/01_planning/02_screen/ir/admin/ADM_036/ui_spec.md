# ADM_036 UI Spec

- screen code: `ADM_036`
- screen name: `판매자정산관리상세(대기)`
- primary route: `/admin/settlements/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `37`

## Screen Requirements

### 1. 정산 상태 안내(정산 대기 중)
  - 정산 처리 가능한 상태임이 고정 노출됨
  - 정산대기 상태는 “딜러 송금 확인 완료 + 판매자 정산 미
  - 처리” 조합으로만 성립됨.
  - 정산 처리 성공 확정 시 해당 상태 안내 영역은 더 이상 노
  - 출 대상이 아니며, 정산완료 화면/상태로 전환됨.

### 2. 거래·판매자·입금 기준 정보
  - 정산 트랜잭션 산정·지급의 기준이 되는 고정 데이터로
  - 노출됨.
  - 차량 정보 : 차량 모델명/트림, 차량 식별 정보(예: 번호
  - 판/차량번호)가 표시됨. 정산 레코드에 “정산 당시 스냅
  - 샷”으로 저장되어야 함
  - 판매자/딜러 식별 : 판매자명, 딜러명(또는 딜러사/담당
  - 자)이 표시됨. 정산 레코드에는 내부 식별자(seller_id,
  - dealer_id)가 함께 고정 저장됨.
  - 판매자 계좌 정보 : 은행명, 계좌번호, 예금주가 표시됨.
  - 본 값은 정산 실행 시 “지급 대상 계좌”로 사용되므로, 화
  - 면 노출값과 서버 지급 대상 값의 불일치가 발생하지 않
  - 도록 정산 실행 직전 서버에서 최신 계좌 검증이 선행됨
  - (계좌 미등록/형식 오류/예금주 불일치 등).
  - 입금(송금) 기준 정보 : 딜러 송금 완료/확인 시각(송금일
  - 시)이 표시됨. 정산 레코드에는 “기준 송금 확인 시각”이
  - 고정 저장되어 감사/정산 분쟁 시 기준 타임스탬프로 사
  - 용됨.

### 3. 정산 금액 상세(정책 10%)
  - 정산 트랜잭션에 사용될 산식 결과가 숫자 단위로 고정
  - 노출됨.
  - 송금 확인 금액 : 딜러 송금 확인 단계에서 확정된 금액이
  - 표시됨. 정산 처리 시 이 값이 “정산 기준 원금”으로 사용
  - 됨. 화면 표시값은 참고이며, 최종 정산 기준 원금은 서버
  - 에서 “송금확인 확정 레코드”를 조회해 고정 적용됨.
  - 플랫폼 수수료(10%) : 정책 수수료율(10%)이 적용된 차
  - 감 금액이 표시됨. 반올림/절사 규칙이 존재하는 경우(원
  - 단위 절사 등) 정산 정책 테이블의 규칙으로 확정 적용되
  - 어야 함.
  - 최종 정산 예정액 : 송금 확인 금액 - 플랫폼 수수료 결과
  - 가 표시됨. 정산 처리 성공 시 실제 지급 요청 금액으로 사
  - 용됨. 정산 처리 버튼 클릭 시, 화면의 표시값이 아니라 서
  - 버 계산 결과가 정산 레코드에 최종 확정 저장됨(프론트
  - 계산값 단독 신뢰 불가).

### 4. 취소
  - 클릭 시 팝업만 종료됨.
  - 거래 상태/정산 상태 변화 없으며, 정산 트랜잭션 호출/정
  - 산 레코드 생성/외부 지급 요청 모두 미발생 상태로 종료
  - 됨.

### 5. 정산 처리
  - 실제 정산(지급) 트랜잭션의 실행 트리거임. 클릭 시 즉시
  - “중복 실행 방지”가 걸려야 하며 동일 거래에 대해 2회 이
  - 상 정산 요청이 승인되지 않도록 서버에서 원자적 잠금이
  - 선행됨.
  - 정산 가능 상태 검증(정산대기 여부), 송금확인 확정 여
  - 부, 판매자 계좌 유효성, 정산 금액(원금/수수료/지급액)
  - 재산정, 처리 권한(관리자 권한) 검증이 통과되어야 함.
  - 정산 레코드가 “처리중”으로 선생성되고, 이후 외부 지급
  - 호출이 발생함. 외부 지급 성공 콜백/응답이 확정되기 전
  - 까지 거래는 “정산대기 유지 + 정산처리중 내부 상태”로
  - 관리됨.
  - 외부 지급 성공이 확정되면 정산 레코드가 “정산완료”로
  - 확정 저장됨. 정산 완료 일시, 처리 관리자, 지급 트랜잭션
  - 식별자(은행/PG 거래번호)가 저장됨. 판매자 정산 상태
  - 가 정산완료로 전환되고, 상위 리스트(판매자 정산 관리)
  - 에서도 상태가 동기 반영됨.
