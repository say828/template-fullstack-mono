# ADM_015 UI Spec

- screen code: `ADM_015`
- screen name: `거래상세(송금정산)`
- primary route: `/admin/remittances/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `16`

## Screen Requirements

### 1. 딜러 → PALKA 송금 영역
  - 송금 금액
  - 딜러가 PALKA 운영 계좌로 송금한 실입금 금액
  - 거래 최종 낙찰가 기준 총 거래 금액
  - 송금 수단
  - 계좌 이체, 가상계좌 등 실제 입금 방식
  - 송금 일시
  - 딜러가 송금을 실행한 시각
  - 거래 재무 흐름 시작 시점
  - 입금 확인 여부
  - PALKA 운영자가 실제 계좌 입금을 확인한 상태 값
  - ‘확인 완료’ 이후에만 다음 정산 단계 진행 가능
  - 입금 확인자
  - 실입금 검증을 수행한 관리자 계정
  - 송금 증빙 보기
  - 딜러 송금 내역 캡처, 계좌 입금 증빙 파일 열람
  - 분쟁·회계 감사 기준 자료
  - 상태 규칙
  - 송금 확인 완료 이후 데이터 불변
  - 미확인 상태에서는 판매자 정산 불가

### 2. PALKA → 판매자 정산 영역
  - 지급 금액
  - 총 거래 금액에서 PALKA 수수료 차감 후 판매자에게
  - 지급되는 실수령 금액
  - 수수료
  - 거래 성사에 따라 확정된 PALKA 운영 수수료
  - 계약·정책 기준 자동 산출
  - 지급 수단
  - 계좌 이체 등 실 지급 방식
  - 지급 일시
  - 판매자 계좌로 실제 입금된 시각
  - 지급 확인 여부
  - 운영자가 판매자 계좌 입금 여부를 검증한 상태
  - 정산 증빙 보기
  - 지급 내역 증빙 문서 열람
  - 상태 규칙
  - 지급 확인 완료 이후 거래 재무 종결 상태 확정
  - 이후 모든 재무 정보 수정 불가

### 3. 정산 요약 영역
  - 총 거래 금액
  - 낙찰가 기준 거래 총액
  - PALKA 수수료
  - 운영 수익 확정 금액
  - 판매자 최종 수령액
  - 판매자에게 귀속된 실 지급 금액
  - 정산 완료 시각
  - 재무 종결 확정 타임스탬프
  - 상태 규칙
  - 본 영역 값은 모든 재무 처리의 최종 기준 데이터

### 4. 정산 로그 영역
  - 거래 재무 단계 전체 이벤트 로그
  - 딜러 송금 실행
  - 입금 확인
  - 판매자 정산 처리
  - 지급 확인
  - 거래 상태 ‘거래 완료’ 전환
  - 모든 로그는 시간순 기록
  - 수정·삭제 불가
