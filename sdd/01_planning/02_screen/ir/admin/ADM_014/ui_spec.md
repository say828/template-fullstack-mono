# ADM_014 UI Spec

- screen code: `ADM_014`
- screen name: `거래상세(인도)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `15`

## Screen Requirements

### 1. 인도 일정 영역
  - 인도 일시
  - 딜러가 확정 등록한 실 인도 실행 시각
  - 인도 단계의 기준 타임스탬프
  - 인도 장소
  - 실물 차량 인도가 수행된 물리적 위치
  - 분쟁·사고 발생 시 검증 기준 위치
  - 인도 방식
  - 직접 인수 / 탁송 등 사전 합의된 전달 방식
  - 이후 분쟁 판단 기준 데이터
  - 담당 기사 / 담당자
  - 인도 실행 책임 주체
  - 문제 발생 시 1차 검증 대상
  - 연락처
  - 담당 딜러의 연락처 표시
  - 상태 규칙
  - 인도 완료 확인 이후 본 영역 데이터는 불변
  - 관리자 포함 전원 수정 불가
  - 인도 관리 이동 > 버튼 클릭 시 (ADM_032) 화면으로 이
  - 동

### 2. 인도 완료 확인 영역
  - 판매자 확인
  - 판매자가 인도 완료를 승인한 시각
  - 실물 이전 완료에 대한 소유권 이전 근거 데이터
  - 딜러 확인
  - 딜러가 인도 완료를 승인한 시각
  - 송금 실행 가능 조건 성립 시점
  - 상태 규칙
  - 양측 확인이 모두 완료되어야 인도 단계 종료
  - 한쪽이라도 미확인 상태일 경우 다음 단계 전환 불가

### 3. 인도 증빙 영역
  - 증빙 자료
  - 인도 과정 촬영 사진, 서류 스캔 등
  - PDF, 이미지 등 증빙 문서
  - 증빙 수량
  - 저장된 증빙 파일 총 개수
  - 증빙 등록 시각
  - 증빙 최종 등록 시각
  - 증빙 상세보기
  - 저장된 모든 증빙 파일 열람
  - 상태 규칙
  - 증빙 파일은 거래 종료 후에도 삭제 불가
  - 법적 분쟁 시 최종 증거 자료

### 4. 인도 타임라인 영역
  - 인도 단계의 전체 이벤트 로그
  - 판매자 인도 확인
  - 딜러 인도 확인
  - 인도 일정 등록
  - 감가 확정 등 이전 단계 이벤트
  - 모든 로그는 시간순 정렬
  - 수정·삭제 불가
