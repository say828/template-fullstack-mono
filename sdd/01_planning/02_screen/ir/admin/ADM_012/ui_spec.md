# ADM_012 UI Spec

- screen code: `ADM_012`
- screen name: `거래상세(감가대기)`
- primary route: `/admin/trades/:vehicleId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `13`

## Screen Requirements

### 1. 딜러 감가 제안 영역
  - 제안 등록 시각
  - 딜러가 감가 제안을 최초 저장한 시각
  - 감가 협의 이력의 기준 시점으로 고정 기록됨
  - 최초 낙찰가
  - 입찰 종료 시 확정된 낙찰가
  - 감가 전 거래 기준가로 고정
  - 감가 총액
  - 감가 항목별 감가 금액의 합계
  - 감가 항목 수정 시 실시간 재산출
  - 최종 제안가
  - 최초 낙찰가 – 감가 총액
  - 판매자 승인 시 최종 거래가로 확정되는 값
  - 딜러 의견
  - 감가 사유에 대한 딜러 서술 코멘트
  - 판매자 판단 근거 데이터로 사용
  - 상태 규칙
  - 본 영역의 데이터는 판매자 승인 전까지 임시 상태
  - 승인 확정 전까지 실제 거래 금액에는 반영되지 않음
  - 딜러 감가 상세보기 > 버튼 클릭 시, 해당하는 거래의 감
  - 가협의상세(ADM_027) 화면으로 이동

### 2. 감가 항목 상세
  - 딜러가 판매자에게 제시한 감가 항목을 보여줌
  - 항목명
  - 사유
  - 감가 금액
  - 합계 행
  - 모든 감가 항목의 감가 금액을 합산하여 표시
  - 상단 감가 총액과 항상 동일 값 유지
  - 감가 항목은 판매자 승인 전까지 딜러가 수정 가능

### 3. 판매자 응답 영역
  - 전달 시각
  - 딜러가 판매자에게 감가 제안을 전송한 시각
  - 경과 시간
  - 현재 시각 기준 응답 경과 시간 실시간 계산
  - 요청 재발송 버튼
  - 관리자가 판매자에게 감가 응답 알림 재전송
  - 상태 변화 없음, 알림 이벤트만 발생
  - 상태 규칙
  - 판매자 승인 전까지 인도·정산 단계 전환 불가

### 4. 감가 협의 로그
  - 감가 협의와 관련된 모든 이벤트 시간순 기록
  - 딜러 감가 등록
  - 판매자 열람
  - 판매자 승인/재협의 요청
  - 재감가 제안
  - 로그는 거래 이력 테이블에 영구 보존
  - 분쟁 발생 시 기준 증빙 데이터로 사용
