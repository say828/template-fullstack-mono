# ADM_039 UI Spec

- screen code: `ADM_039`
- screen name: `판매자상세(기본탭)`
- primary route: `/admin/sellers/:sellerId`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`
- page number: `40`

## Screen Requirements

### 1. 판매자 식별 요약(헤더)
  - 패널의 기준 대상이 되는 판매자 1명을 고정 식별하는 영
  - 역.
  - 노출값: 판매자명 + 상태 태그(정상/블랙리스트/탈퇴 중
  - 1) + 판매자ID.
  - 본 패널의 모든 정보(3~5번) 및 운영 액션(6번)은 1번의
  - 판매자ID에 종속되는 구조로 고정.
  - 상태 태그는 단순 표시가 아니라 “목록 필터의 상태 값”과
  - 동일 기준으로 정합 유지 필요.

### 2. 상세 탭(정보 범위 전환)
  - 동일 판매자에 대한 상세 정보를 “기본 및 정산 정보 / 차
  - 량 판매 이력” 범위로 전환하는 탭.
  - 탭 전환은 동일 패널 내 콘텐츠만 변경되며, 판매자ID/상
  - 태(1번)는 고정 유지.
  - 기본 진입 탭은 “기본 및 정산 정보”로 고정 전제.
  - “차량 판매 이력(3)”의 숫자는 해당 판매자 기준 판매 이
  - 력 건수 요약값으로 노출
  - 차량판매이력 탭 클릭 시 (ADM_040) 화면으로 탭 전환

### 3. 기본 정보(계정 메타 정보)
  - 운영 판단을 위한 계정의 핵심 메타 정보를 고정 노출하
  - 는 영역
  - 노출 필드(이미지 기준): 연락처, 이메일, 가입 경로, 거주
  - 국가, 가입일, 최근 로그인.
  - “가입 경로”는 목록의 가입방식과 동일 의미로 정합 유지
  - “최근 로그인”은 계정 접근성/휴면 판단 근거로 사용되므
  - 로 날짜 포맷 일관성 고정.

### 4. 정산 계좌 정보
  - 판매자의 정산 수령 계좌가 등록된 경우에만 정보가 의미
  - 있게 채워지는 영역.
  - 계좌 등록 완료: 은행명 + 계좌번호 + 예금주 노출.
  - 계좌 미등록: 동일 영역은 유지하되 값은 공란/미등록
  - 상태로 노출.
  - 계좌 정보는 “정산 실행 가능 여부”의 선행 확인 정보로
  - 취급됨

### 5. 통장사본 이미지(증빙 뷰어)
  - 판매자가 제출한 통장사본을 이미지로 열람하는 영역.
  - 뷰어 기능은 “열람 보조” 범위로 마우스 호버시 노출: 확
  - 대/축소, 원본 크기 보기, 회전/리셋, 다운로드 등의 컨트
  - 롤 제공(아이콘 기반).

### 6. 블랙리스트 등록
  - 선택 판매자를 “블랙리스트 상태”로 전환시키는 운영 액
  - 션 버튼.
  - 노출/활성 규칙(상태 기반 분기 필요)
  - 현재 상태=정상: 버튼 활성(등록 가능).
  - 현재 상태=블랙리스트: 버튼명 “블랙리스트 해제” 로
  - 변경됨
  - 현재 상태=탈퇴: 버튼명 “닫기”로 변경됨
  - 처리 결과
  - 등록 완료 시 1번 상태 태그 즉시 “블랙리스트”로 갱
  - 신.
  - 목록 화면으로 복귀 시 해당 판매자의 상태가 “블랙리
  - 스트”로 반영되어 필터/검색 결과에 즉시 영향.
  - 클릭 시 블랙리스트 (ADM_041) 화면 호출
