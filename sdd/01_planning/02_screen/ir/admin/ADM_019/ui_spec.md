- primary route: `/admin/inspections`

- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`

## Screen Requirements

### 1. 검차 운영 조회/필터 영역
  - 상단 필터는 “검차 운영 대상 거래”를 운영 상태 기준으로
  - 분류·탐색하기 위한 제어군으로 구성됨.
  - 상태는 본 화면 목록의 운영 큐(Queue) 분류 기준이며,
  - 집계 카드(2)의 카테고리와 동일한 분류 체계를 사용함
  - (예: 일정 제안 필요/응답 대기/조율 요청/확정/검차 진
  - 행/지연).
  - 검차 상태는 검차 단계의 실제 진행 상태(검차 단계 내 세
  - 부 상태) 필터이며, 운영 큐(상태)와 독립적으로 적용됨
  - (운영자는 “지연” 큐 안에서 “리포트 미제출”만 재필터링
  - 하는 식의 교차 탐색이 가능해야 함).
  - 지역은 거래의 검차 수행 지역(검차 장소 기준) 필터로 동
  - 작하며, 국가/도시 체계는 운영 정책(국내/해외)에 따라
  - 서버 제공 목록을 그대로 사용함.
  - 평가사는 배정된 평가사 기준 필터로 동작하며, 배정 전
  - 거래는 “미배정” 값으로 조회 가능해야 함.
  - 기간은 “검차 운영 이벤트 기준 기간”으로 동작하며, 기본
  - 값은 최근 7일로 로드됨. 이 기간 기준은 집계(2)와 목록
  - (3)에 동일 적용됨.
  - 검색은 거래 식별과 빠른 운영 접근을 위해 제공되며, 최
  - 소 거래ID / 차량번호 / 판매자명 / 딜러명을 단일 입력으
  - 로 통합 검색함.
  - 초기화는 본 화면의 모든 필터/검색 조건을 기본값으로
  - 복원하며, 복원 즉시 목록과 집계가 동일 기준으로 재조
  - 회됨.

### 2. 검차 운영 집계(상태별 운영 큐 현황)
  - 집계 카드는 현재 조회 조건(1)이 적용된 결과 중, 운영자
  - 가 즉시 처리 우선순위를 판단할 수 있도록 검차 운영 큐
  - 별 대기 건수를 요약 표시함.
  - 일정 제안 필요는 검차 일정이 아직 제안되지 않아 다음
  - 단계(조율/확정)로 진행 불가한 건수로 집계됨
  - 응답 대기는 검차 일정 제안이 발송되었으나, 상대 주체
  - 의 응답이 확정 조건을 충족하지 못해 대기 중인 건수로
  - 집계됨.
  - 조율 요청은 일정 재조율 등의 사유로 추가 조율을 수행
  - 해야 하는 상태의 건수로 집계됨.
  - 확정은 검차 일정이 확정되어 예정대로 진행 가능하며,
  - 리포트 제출 전 단계의 추적 대상으로 집계됨.
  - 검차 진행은 검차 수행이 시작되었거나(또는 수행 중으로
  - 전환된) 결과/리포트 제출 전 상태의 건수로 집계됨.
  - 지연은 SLA 기준을 초과한 상태(예: 확정 후 일정 미이행,
  - 수행 후 리포트 미제출 등)로 집계되며, 목록의 SLA 컬럼
  - (3)과 동일한 산식으로 판정되어야 함.
  - 각 카드의 숫자는 “현재 시점 기준 실시간 집계”이며, 필
  - 터 변경 시 즉시 재계산되어 표시됨.

### 3. 검차 운영 대상 목록 영역
  - 목록은 “검차 운영 대상 거래”를 행 단위로 노출하며, 각
  - 행은 거래 1건 = 운영 단위 1건으로 고정됨
  - 거래ID는 거래의 고유 식별자이며 클릭 시 검차 운영 상
  - 세(ADM_020 등) 화면으로 이동.
  - 차량 정보는 운영 식별을 위한 최소 요약(연식/모델/트
  - 림/주행거리 등)으로 구성되며, 차량 상세의 진실 소스는
  - 거래에 연결된 차량 마스터 데이터임.
  - 판매자/딜러는 거래 당사자 표시이며, 검차 단계에서 책
  - 임 주체 판단(응답 대기/조율 요청)과 연결됨.
  - 지역은 검차 수행 지역이며, 1번 지역 필터의 기준 필드와
  - 동일해야 함.
  - 검차 일정은 확정된 일정이 존재하면 확정 일시를 표시하
  - 고, 확정 전이면 제안/대기 상태에 따라 비어 있거나(또는
  - “미정”)로 표시됨.
  - 평가사는 배정된 평가사명을 표시하며, 미배정이면 “-”로
  - 표시됨(미배정 상태는 집계/필터에서 운영 우선순위로
  - 활용됨).
  - 상태는 본 화면의 운영 큐 태그이며, 집계(2) 및 필터(1)
  - 의 상태와 동일한 값 체계를 사용함.
  - 검차상태는 검차 단계 내 세부 진행 태그이며, 운영 큐와
  - 별개로 표시되어 “지연 사유가 일정/리포트 중 무엇인
  - 지”를 판별하는 힌트로 사용됨.
  - 리포트는 검차 리포트의 제출/검증 상태를 표시함
  - SLA는 검차 운영 기준 마감 대비 잔여/초과를 D- / D+로
  - 표시함
  - 최종 업데이트는 본 거래의 검차 관련 이벤트(일정 제안/
  - 확정/상태 변경/리포트 제출 등) 기준 마지막 변경 일자
  - 를 표시하며, 운영 판단의 최신성 기준으로 사용됨.
