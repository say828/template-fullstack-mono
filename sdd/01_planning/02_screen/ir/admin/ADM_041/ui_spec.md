- primary route: `/admin/blacklist`
- access: `role:ADMIN`
- source spec: `sdd/01_planning/02_screen/ir/admin/admin_screen_spec.pdf`

## Screen Requirements

### 1. 사유 선택
  - 블랙리스트 처리의 1차 분류값 선택 영역.
  - • 선택값은 블랙리스트 사유의 대표 분류로 저장됨(통
  - 계/필터/로그 기준값)
  - 미선택 상태에서는 등록 확정 불가 상태 유지

### 2. 상세 사유 입력
  - 운영자가 실제 제재 근거를 문장으로 확정 기록하는 영
  - 역.
  - 입력값은 제재 근거의 원문 기록으로 저장되며, 이후 감
  - 사/분쟁/재검토 시 우선 참조 데이터로 취급
  - 최소 입력 길이 정책 필요(예: 20자/50자 등) 및 공백만
  - 입력 방지 필요

### 3. 제재 기간 선택
  - 블랙리스트 상태의 유효 기간을 확정하는 선택 영역.
  - 기간 선택값은 제재 종료 시점을 산정하는 기준값으로 저
  - 장됨
  - “무기한/기간 프리셋(7일, 30일 등)” 중 서비스 정책에
  - 맞는 옵션으로 고정 제공 필요
  - 종료 시점 도달 시 자동 해제 대상이 되며, 자동 해제 시점
  - 은 시스템 공통 기준시(예: 등록 시각 기준)

### 4. 취소
  - 제재 등록 작업 중단 액션.
  - 클릭 시 입력/선택 값 저장 없이 팝업 종료
  - 팝업 재진입 시 초기 상태로 로딩

### 5. 블랙리스트 등록
  - 제재 등록 확정 트리거.
  - • 1,2,3 입력/선택값 유효성 충족 시에만 활성 상태
  - 등록 확정 시 판매자 상태가 블랙리스트로 전환되고, 판
  - 매자 상세 화면의 버튼 문구가 “블랙리스트 해제”로 변경
  - 됨
  - 등록 확정 시점의 사유/상세사유/기간/처리자/처리일시
  - 가 블랙리스트 이력으로 남는 기준 데이터가 됨
  - 제재 기간 종료 시 자동 해제되며, 자동 해제 직후 판매자
  - 상태가 정상으로 복구되고 버튼이 “블랙리스트 등록”으
  - 로 복구됨
