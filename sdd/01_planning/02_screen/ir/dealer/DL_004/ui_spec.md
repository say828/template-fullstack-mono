# DL_004 UI Spec

- screen code: `DL_004`
- screen name: `매물상세(정보)`
- primary route: `/dealer/market/:vehicleId`
- access: `role:DEALER`
- source spec: `sdd/01_planning/02_screen/ir/dealer/dealer_screen_spec.pdf`
- page number: `5`

## Screen Requirements

### 1. 매물 핵심 정보 영역
  - 매물의 현재 입찰 상태(입찰중 / 마감임박), 거래 유형(국
  - 내거래 / 수출가능), 대표 이미지 노출
  - 입찰 상태는 서버 기준 실시간 반영되며, 마감 임박 상태
  - 에서는 남은 시간 카운트다운 기준으로 자동 전환 표시
  - 본 영역은 딜러의 거래 참여 가능 여부 1차 판단 지점으
  - 로 사용

### 2. 차량 기본 정보 요약
  - 차량명, 연식, 주행거리, 연료, 변속기, 차량번호, 사고 이
  - 력, 소유 형태, 판매자 유형 등 거래 성립에 직접 영향을
  - 주는 핵심 판단 데이터 집계 노출
  - 해당 정보는 이후 입찰·검사·정산 모든 프로세스의 기준
  - 데이터로 참조됨

### 3. 차량 상세 정보 탭
  - 기본 사항 / 사진·영상 / 검사·감가 안내 / 거래 조건 탭 구
  - 성
  - 딜러가 차량 상태·검사 결과·거래 제약 조건을 단계적으
  - 로 검토하는 구조
  - 사진영상 탭 클릭 시 (DL_005) 화면으로 탭 전환
  - 검차/감가안내 클릭 시 (DL_007) 화면으로 탭 전환
  - 거래조건 클릭 시 (DL_008) 화면으로 탭전환

### 4. 입찰 현황 및 입력 영역
  - 현재 최고 입찰가, 입찰 참여 건수, 마감 잔여 시간 실시간
  - 표시
  - 딜러가 본인 입찰 금액 직접 입력
  - • 숫자만 입력 가능
  - - 판매자가 정한 최소 입찰 단위 기준 충족 필수
  - - 유효하지 않은 금액 입력 시 즉시 경고 메시지 노출 및
  - 입찰 불가 처리

### 5. 입찰 실행
  - 입찰하기 클릭 시 최종 입찰 검증 수행 후 입찰 참여 처리
  - 마감 이전 + 유효 금액 조건 충족 시 입찰 성공 → 최고 입
  - 찰가 및 입찰 건수 즉시 갱신
