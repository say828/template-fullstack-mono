# DL_023 UI Spec

- screen code: `DL_023`
- screen name: `감가금액입력`
- primary route: `/dealer/transactions/:vehicleId/depreciation`
- access: `role:DEALER`
- source spec: `sdd/01_planning/02_screen/ir/dealer/dealer_screen_spec.pdf`
- page number: `24`

## Screen Requirements

### 1. 감가 항목 입력 영역
  - 딜러가 감가 사유를 항목 단위로 구조화하여 제안하는 핵
  - 심 입력 영역
  - 각 항목은 개별 감가 사유로 서버에 저장되며, 모든 항목
  - 의 합산값이 거래 금액 조정의 근거됨.
  - 각 행은 독립적인 감가 객체로 저장됨
  - 항목·내용·감가금액 중 하나라도 누락되면 해당 항목 전
  - 체 무효 처리
  - 감가 금액은 0 초과 정수, 소수·음수·문자 입력 불가
  - 행 삭제 시 해당 항목은 서버 전송 대상에서 제외됨
  - 감가 항목 금액 변경 시 총 감가 금액 및 최종 거래 예상
  - 금액 즉시 재계산
  - 최종 거래 예상 금액은 항상 낙찰가 이상이 될 수 없음

### 2. 항목 추가
  - 클릭 시 동일 구조 입력 행 동적 생성
  - 최대 10개 항목까지 추가 가능
  - 10개 초과 시 더 이상 생성 불가

### 3. 첨부 파일
  - 감가 근거 증빙 자료 업로드
  - 최대 5개, 파일당 10MB, 형식: PDF / JPG / PNG
  - 서버 저장 후 감가 제안 데이터와 함께 판매자에게 전달

### 4. 감가 사유
  - 감가 협의 전체 사유 서술 영역
  - 미입력 시 제출 불가
  - 판매자 승인 판단의 핵심 근거 데이터

### 5. 금액 계산 영역
  - 낙찰가: 경매 종료 시 확정된 원 거래 금액
  - 총 감가 금액: 모든 감가 항목 금액 합산 (항상 음수 표현)
  - 최종 거래 예상 금액 = 낙찰가 − 총 감가 금액
  - 계산은 모든 입력 변경 시 즉시 반영, 서버 제출 전 클라이
  - 언트·서버 양측에서 동일 로직 검증 수행.

### 6. 취소
  - 클릭 시 모든 입력값은 폐기 되며, 팝업 종료
  - 거래 상태 변화 없음

### 7. 제안하기
  - 딜러의 감가 협의 요청을 거래 흐름에 반영하는 결정 버
  - 튼
  - 클릭 시 입력 데이터 서버 검증 후 감가 제안 데이터 저장
  - 처리
  - 거래 상태 → 감가 협의 대기
  - 판매자에게 감가 협의 알림 발송
  - 거래 화면의 다음 단계 활성화
