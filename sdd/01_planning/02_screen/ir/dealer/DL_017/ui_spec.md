# DL_017 UI Spec

- screen code: `DL_017`
- screen name: `거래결제(진행중)`
- primary route: `/dealer/transactions`
- access: `role:DEALER`
- source spec: `sdd/01_planning/02_screen/ir/dealer/dealer_screen_spec.pdf`
- page number: `18`

## Screen Requirements

### 1. 거래 상태 집계 영역
  - 딜러 계정 기준 거래 데이터를 진행중 / 정산대기 / 완료
  - 로 실시간 집계하여 표시
  - 집계 수치는 현재 딜러의 유효 거래 레코드 기준으로 산
  - 출됨
  - 상태 변경 발생 시 즉시 갱신됨

### 2. 거래 필터 영역
  - 거래 단계 필터: 검차 일정 조율 / 검차완료 / 감가 협의 /
  - 인도 진행 / 송금·정산
  - 거래 유형, 기간, 정렬, 검색 조건이 복합 적용됨
  - 필터 변경 시 서버 조회 기준 즉시 반영됨

### 3. 거래 탭 영역
  - 진행 중 거래 / 완료된 거래로 거래 데이터 분리 표시
  - 탭 변경 시 거래 상태 기준 자동 필터링됨
  - 완료된 거래 탭 클릭 시 (DL_018) 화면으로 탭 전환

### 4. 거래 리스트 영역
  - 각 행은 하나의 낙찰 확정 거래 단위이며, 해당 거래의 현
  - 재 처리 단계와 다음 분기 화면을 결정하는 기준 정보를
  - 포함함.
  - 거래번호: 낙찰 확정 시 생성되는 거래 고유 식별값
  - 매물 정보: 차량 요약 정보
  - 거래 유형: 국내 거래 / 해외(수출)
  - 낙찰가: 최종 낙찰 금액
  - 감가 확정가: 감가 협의 종료 후 확정 금액
  - 현재 단계: 거래 처리 단계 값
  - 상태: 각 단계의 세부 진행 상태
  - 액션: 다음 처리 화면으로 이동하는 제어 버튼

### 5. 거래·결제 진행 버튼
  - 클릭 시 해당 거래 행의 현재 단계 값 기준으로 다음 화면
  - 분기
  - 각 단계에 따라 클릭 시 해당하는 거래결제 상세 화면으
  - 로(DL_020 등) 이동
