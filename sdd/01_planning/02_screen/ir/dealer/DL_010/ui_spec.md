# DL_010 UI Spec

- screen code: `DL_010`
- screen name: `나의입찰`
- primary route: `/dealer/bids`
- access: `role:DEALER`
- source spec: `sdd/01_planning/02_screen/ir/dealer/dealer_screen_spec.pdf`
- page number: `11`

## Screen Requirements

### 1. 상단 요약 영역
  - ‘입찰중/낙찰/미낙찰’ 3개 상태만 KPI로 집계하여 상단
  - 배지로 고정 노출됨.
  - 집계 기준은 “딜러 계정이 입찰을 생성한 건”
  - 상태 변경(마감 도래, 낙찰 확정 등) 시 본 화면 재진입/새
  - 로고침 시점에 즉시 반영됨.

### 2. 상태 탭/필터 영역
  - 탭은 “전체/입찰중/마감/낙찰/미낙찰/취소됨”으로 구성
  - 됨.
  - 탭은 단순 UI 분류가 아니라 행의 컬럼 값 해석과 액션 노
  - 출 규칙을 강제하는 분기 기준으로 사용됨.
  - 상단 드롭다운 필터(거래유형, 기간, 정렬 등)와 검색은
  - 선택 조합에 따라 결과 리스트만 갱신되며, 집계(1번)는
  - 필터 적용 기준으로 갱신됨

### 3. 입찰 리스트 행(상태별 데이터 규칙)
  - 공통 구성: 썸네일/차량명/연식·주행거리·연료 등 요약 +
  - 거래유형 + 내 입찰가 + 현재 최고 입찰가 + 입찰 마감 +
  - 상태 태그 + 액션.
  - 상태=입찰중
  - • ‘현재 최고 입찰가’는 실시간 최고가 기준으로 노출됨.
  - • ‘내 입찰가’ 대비 결과 힌트가 함께 노출됨(예: 추월됨/
  - 최고가). 기준은 “내 입찰가 < 현재 최고 입찰가”이면 추
  - 월됨, “=”이면 최고가로 표기됨.
  - • ‘입찰 마감’은 D-1/D-2처럼 상대값 + 마감일자(또는 시
  - 간) 병기 가능하며, 리스트 우선순위는 마감 임박 순 또는
  - 최신 참여 순(선택된 정렬값 기준).
  - 상태=마감(결과 확정 전)
  - • 입찰 마감이 도래했으나 낙찰/미낙찰이 아직 확정되지
  - 않은 중간 상태로 정의됨.
  - • ‘입찰 마감’ 컬럼은 “마감됨(일시)”로 고정 노출되고, 상
  - 대값(D-x)은 미노출됨(이미 종료된 이벤트이므로).
  - • ‘상태’는 “마감” 태그로 고정되며, 액션은 “상세보기”만
  - 유지됨(수정/거래진입 모두 불가).
  - • 본 상태는 백오피스/시스템 판정에 의해 낙찰/미낙찰로
  - 전환됨.
  - 상태=낙찰
  - • 낙찰 확정 후 상태 태그가 “낙찰”로 전환됨.
  - • ‘현재 최고 입찰가’는 낙찰가와 동일 값으로 간주되며,
  - 내 입찰가와 동일 여부가 명확해야 함(낙찰은 곧 내 입찰
  - 가=최종 확정가).
  - • 후속 프로세스 진입이 가능해야 하므로, 행의 액션은 거
  - 래·결제 진입 버튼이 우선 노출됨.
  - 상태=미낙찰
  - • ‘미낙찰’은 입찰 결과가 확정되었고 딜러가 낙찰에 실패
  - 한 상태임.
  - • 금액 컬럼은 조회 목적만 남고(내 입찰가/최고가), 액션
  - 은 “유사 매물 보기”를 우선 제공하여 재참여 전환을 유도
  - 함.
  - 상태=취소됨
  - • 딜러에 의해 입찰이 취소되었거나, 정책상 입찰이 무효
  - 처리된 상태로 정의됨.
  - • 상태 컬럼은 “취소됨” 태그로 고정되고, 액션은 상세보
  - 기만 제공됨

### 4. 입찰가 수정(상태=입찰중 전용)
  - 상태가 ‘입찰중’인 행에서만 노출됨.
  - 클릭 시 해당 매물의 입찰가 수정(DL_011)화면을 호출하
  - 여 금액 재입력을 수행함.
  - 수정 가능 범위는 “마감 전”이며, 마감 시각 도달 즉시 수
  - 정 트리거 자체가 차단됨(비노출)
  - 수정 완료 시 본 리스트의 ‘내 입찰가’와 ‘추월됨/최고가’
  - 표기가 즉시 재계산되어 반영됨

### 5. 상세보기(상태별 진입 화면 분기)
  - 모든 상태에서 노출되며, 클릭 시 매물 상세로 이동하되
  - 진입 목적/가능 행위가 상태별로 달라짐.
  - 상태=입찰중: 클릭 시 입찰상세(입찰중)화면(DL_012)
  - 으로 이동함
  - 상태=마감: 상세 화면은 결과 확정 전 안내 컨텍스트(예:
  - “결과 확인 중”)로 고정되며, 클릭 시 입찰상세(마감) 화
  - 면(DL_013)으로 이동함
  - 상태=낙찰: 클릭 시 입찰상세(낙찰) 화면(DL_015) 으로
  - 이동함
  - 상태=미낙찰: 상세 화면은 조회 전용이며, 클릭 시 입찰상
  - 세(미낙찰)화면(DL_014)으로 이동함
  - 상태=취소됨: 클릭 시 입찰상세(취소됨)화면(DL_016)으
  - 로 이동함

### 6. 유사 매물 보기(상태=미낙찰 전용)
  - 미낙찰 행에서만 노출됨.
  - 클릭 시 유사 조건(차종/연식/주행거리/거래유형 등) 기
  - 반 추천 리스트(DL_003)로 이동됨

### 7. 거래·결제 진행(상태=낙찰 전용)
  - 낙찰 행에서만 노출됨.
  - 클릭 시 “거래·결제” 화면(DL_020)으로 이동하여 후속
  - 단계(대금 송금, 계약/서류, 인도/정산 등) 프로세스를 시
  - 작함.
  - 진입 시점부터 해당 건은 “딜러의 실제 거래 의무가 발생
  - 한 상태”로 취급됨
