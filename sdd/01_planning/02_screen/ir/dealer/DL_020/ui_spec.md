# DL_020 UI Spec

- screen code: `DL_020`
- screen name: `거래결제상세(검차일정)`
- primary route: `/dealer/transactions/:vehicleId/detail/inspection`
- access: `role:DEALER`
- source spec: `sdd/01_planning/02_screen/ir/dealer/dealer_screen_spec.pdf`
- page number: `21`

## Screen Requirements

### 1. 매물 상세 보기 이동
  - 해당 거래에 연결된 매물 상세 화면으로 이동하는 단축
  - 링크.
  - 거래 이행 도중 차량 정보·사진·검차 조건을 재확인하기
  - 위한 보조 진입 경로.
  - 클릭 시 해당하는 매물의 상세 (DL_015) 화면으로 이동

### 2. 거래 단계 진행 바
  - 현재 거래의 이행 단계를 시각적으로 표현하는 단계 인디
  - 케이터.
  - 현재 단계가 2단계(검차 일정)에 고정 표시되며, 단계 이
  - 동은 딜러의 직접 조작이 아닌 시스템 이벤트에 의해 자
  - 동 전환됨.

### 3. 낙찰 정보 영역
  - 해당 거래의 법적·계약적 기준 데이터 영역.
  - 낙찰가: 최종 확정된 거래 금액
  - 낙찰 일시: 낙찰 확정이 서버에 기록된 시점
  - 거래 유형: 국내 거래 / 해외(수출)
  - 판매자: 낙찰 대상 매물의 소유자 정보
  - 해당 데이터는 이후 모든 단계의 기준값으로 수정 불가
  - 상태로 유지된다.

### 4. 거래 단계 상세 안내 영역
  - 각 단계별 실제 업무 프로세스를 딜러가 이해할 수 있도
  - 록 단계별 상태 설명을 제공하는 고정 정보 영역.
  - 현재 단계: 검차 일정 조율
  - PALKA가 판매자·평가사와 협의하여 검차 일정 조율 중
  - 검차 일정 확정 시, 다음 단계(검차 완료)로 자동 전환

### 5. 거래 요약 패널
  - 딜러가 거래 전체 현황을 빠르게 파악하기 위한 요약 카
  - 드.
  - 차량 정보: 차종 및 차량번호
  - 낙찰가: 최종 거래 금액
  - 최종 거래가: 감가 협의 확정 이후 반영될 예정 금액
  - 진행 상태: 현재 단계 상태값 (낙찰 완료)

### 6. 고객센터 이동
  - 거래 진행 중 발생하는 모든 분쟁·이슈에 대해 고객센터
  - 문의 화면으로 즉시 이동하는 지원 링크.
  - 클릭 시 1:1 문의(DL_038) 화면으로 이동
