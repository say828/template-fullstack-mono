# DL_011 UI Spec

- screen code: `DL_011`
- screen name: `입찰가수정`
- primary route: `/dealer/bids/:vehicleId/edit`
- access: `role:DEALER`
- source spec: `sdd/01_planning/02_screen/ir/dealer/dealer_screen_spec.pdf`
- page number: `12`

## Screen Requirements

### 1. 매물 요약 영역
  - 수정 대상 매물의 썸네일, 차량명, 연식, 주행거리, 연료,
  - 현재 최고 입찰가를 고정 표시함.
  - 해당 정보는 수정 대상 입찰의 기준 정보이며, 팝업 오픈
  - 시 서버 기준 최신 데이터로 로딩됨.
  - ‘현재 최고가’는 수정 입력값의 최소 허용 기준 판단에 사
  - 용됨.

### 2. 수정 입찰가 입력 영역
  - 딜러가 변경할 입찰 금액을 직접 입력하는 핵심 입력 필
  - 드임.
  - 숫자만 입력 가능하며, 천 단위 콤마 자동 적용됨.
  - 입력창 하단의 +10만 / +50만 / +100만 / +500만 버튼
  - 클릭 시 현재 입력값에 해당 금액이 누적 가산되며, 가산
  - 후에도 위 모든 유효성 검증 규칙이 즉시 적용됨.

### 3. 수정 내용 확인 동의
  - 입찰가 수정에 따른 경쟁 구조 변경 및 최종 결과 영향을
  - 인지했음을 확인하는 필수 동의 체크 항목임.
  - 체크하지 않으면 수정 완료 버튼은 비활성 유지됨

### 4. 취소 버튼
  - 클릭 시 수정 내용 저장 없이 팝업 닫힘.
  - 입력한 임시 값은 모두 폐기되며 기존 입찰가는 변경되지
  - 않음

### 5. 수정 완료 버튼
  - 모든 유효성 검증 및 동의 조건이 충족된 경우에만 활성
  - 화됨.
  - 클릭 시 서버에 입찰 수정 요청 전송 → 성공 시 현재 입
  - 찰 상태 즉시 갱신됨.
  - 수정 성공 시 본 팝업 종료 후 나의 입찰 리스트 및 매물
  - 상세의 모든 관련 금액·상태가 동기화 갱신됨.
  - 입찰가가 수정되었습니다.
- 40,500,000원
  - 으로 정상적으로 수정되었습니다.
  - 확인
