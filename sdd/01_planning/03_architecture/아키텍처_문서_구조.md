# 아키텍처 문서 구조

## Scope

- `template`는 `client/*`, `server/*`, `sdd/*`를 중심으로 운영하는 monolithic delivery repo다.
- backend owner는 `server/contexts/*` DDD context다.
- frontend owner는 `client/web`, `client/admin` surface다.
- 인프라 planning은 AWS ingress + OpenStack backend/runtime 경계로 설명한다.

## Structure Rule

- 공통 구조와 경계는 이 문서와 하위 README에서 유지한다.
- 화면 수준 상호작용은 `sdd/01_planning/02_screen/`과 `sdd/02_plan/02_screen/`에서 관리한다.
- API 계약은 `sdd/01_planning/05_api/`, 데이터 모델은 `sdd/01_planning/04_data/`에서 관리한다.

## Current Baseline

- backend는 `identity`, `dealers`, `vehicles`, `bidding`, `trades`, `settlement`, `support`, `settings`, `admin` context를 가진 monolith다.
- 각 context는 `domain/application/infrastructure/presentation` 계층을 따른다.
- frontend는 customer-facing `client/web`와 관리자 콘솔 `client/admin`으로 구분한다.
- 배포 경계는 `AWS frontend edge -> OpenStack backend/runtime` 기준으로 유지한다.
